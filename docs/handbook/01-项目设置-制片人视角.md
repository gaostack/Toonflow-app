# 第01章 项目设置：用制片人的脑子填好每一个字段

## 本章导览

很多人第一次用 Toonflow，会把"创建项目"当成一个登记表——随手填个名字、选个画风，点确定，然后一头扎进对话框里抱怨"AI 怎么不懂我"。

这是新人最贵的一课。

在真实的影视项目里，**立项书决定了片子的天花板**。剧本能写多准、分镜能拍多稳，70% 在你按下"创建"那一刻就定了。Toonflow 的创建项目表单，本质上就是一份压缩到极致的立项书：它直接喂给两个 AI Agent（剧本 Agent、制作 Agent），是它们理解"你到底想要什么片子"的唯一入口之一。

本章我们逐字段拆解 `src/routes/project/addProject.ts` 里的每一个参数，告诉你：

- 每个字段**真正控制什么**、**实际进了哪个 Agent 的脑子**（这点至关重要，决定了"想改 X 该回头调哪个字段"）；
- 哪些字段是"创作灵魂"（重点是 **小说简介** 和 **视觉手册/导演手册**），怎么写出制片级别的质量；
- 不同题材的推荐配置样例，可以直接抄。

先看一眼字段全貌（来自 `o_project` 表，定义在 `src/lib/initDB.ts`）：

| 字段 | 含义 | 进剧本 Agent？ | 进制作 Agent？ |
|---|---|---|---|
| `name` | 小说/项目名称 | ✅（项目语境） | — |
| `type` | 小说类型 | ✅（项目语境） | — |
| `intro` | 小说简介 | ✅（项目语境） | — |
| `artStyle` | 视觉手册 / 画风 | ✅（项目语境，作"视觉参考"） | ✅（贯穿资产、分镜、视频生成） |
| `directorManual` | 导演手册 | ❌ **不进** | ✅（制作 Agent 调度依据） |
| `videoRatio` | 视频画幅 | ✅（项目语境） | ✅（出图/出片尺寸） |
| `imageModel` | 图像模型 | ❌ | ✅ |
| `videoModel` | 视频模型 | ❌ | ✅ |
| `imageQuality` | 图像清晰度 | ❌ | ✅ |
| `mode` | 视频参考模式 | ❌ | ✅（决定提示词模式） |
| `projectType` | 项目类型标识 | — | — |

> **关键认知**：剧本 Agent 看不到模型字段，也看不到导演手册；制作 Agent 才是真正消费画风/导演/模型/画幅这些"制作参数"的人。记住这张表，后面所有"我想改某个效果该调哪里"的问题都能秒回。

## 一、小说名称 `name`

### 它控制什么
项目的显示名，同时作为"小说名称"出现在剧本 Agent 的项目语境里（见 `src/agents/scriptAgent/index.ts` 拼接的 `projectInfo`）。

### 为什么重要
AI 会把名字当成**题材与基调的第一信号**。`《重生之都市仙尊》` 和 `《林川的二三事》`，模型对前者的世界观假设（修真、扮猪吃虎、打脸爽点）会比后者强烈得多。名字是廉价但有效的"语境锚"。

### 怎么填
- 用**原著真实书名**，别用项目代号（`项目A`、`test1` 这种是在浪费一个免费的语境信号）。
- 如果原著名太隐晦，可在名字里轻微点题，但不要堆砌。

| | Before | After |
|---|---|---|
| 反例 | `新项目` / `测试3` | — |
| 正例 | — | `《我在末世捡空投》` |

**常见错误**：用纯英文/纯编号命名，让 AI 完全 get 不到题材；或者名字与简介里的设定打架（名字写"仙侠"，简介写的是赛博朋克）。

## 二、小说类型 `type`（被严重低估的字段）

### 它控制什么
进剧本 Agent 项目语境的 `小说类型：xxx`。它是 AI 选择**叙事节奏、爽点密度、人物关系套路**的总开关。

### 为什么重要
类型是行业的"通用语"。你写"都市逆袭爽文"，AI 就知道：开篇要快、要有压制、要有打脸、单集要留钩子；你写"群像年代剧"，它就会放慢节奏、铺人物。类型填得越精确，AI 改编时的"默认假设"越接近你要的东西。

### 怎么写（精确度三档）
- **及格**：一个大类。`都市`、`悬疑`、`古风`。
- **良好**：大类 + 核心套路。`都市 / 战神归来 / 隐藏身份打脸`。
- **优秀**：大类 + 套路 + 受众基调。`古风仙侠 / 双男主救世 / 偏热血与宿命感，受众偏年轻女性`。

> 提示：`type` 是纯文本，不是枚举。你完全可以写一句话，把"它是什么 + 它的调性"说清楚。AI 吃的是语义，不是标签。

| | Before | After |
|---|---|---|
| 反例 | `小说` | `都市` |
| 良好 | `都市` | `都市爽文 / 赘婿逆袭 / 强打脸节奏` |
| 优秀 | — | `都市爽文 / 赘婿逆袭，主线是隐藏富豪身份被羞辱后逐步亮剑；基调爽快、节奏快、单集一个反转钩子` |

**常见错误**：
1. 留空或写"小说"——等于没填。
2. 类型与简介脱节——`type` 写"轻喜剧"，`intro` 却是悲情虐恋，AI 会被两种基调撕扯，产出忽冷忽热。
3. 把"题材"当"画风"写（`type` 写"二次元风格"）——画风是 `artStyle` 的活，别串台。

## 三、小说简介 `intro`（本章最重要的字段，请认真读）

### 它控制什么
进剧本 Agent 项目语境的 `小说简介：xxx`。这是**整个剧本改编方向的总纲**。骨架阶段虽然喂的是逐章事件表，但 AI"为什么这样取舍、往哪个高潮收束、谁是主角弧线核心"——靠的就是这段简介。

### 为什么重要（讲透）
请把 `intro` 当成你递给一个金牌编剧的**一页纸立项书**。编剧拿到原著一百万字没空全读，他先读你这一页，决定"主线抓什么、爽点埋哪里、人物钩子是谁"。

简介写得像豆瓣一句话剧情梗概，AI 改出来的剧本就是"什么都讲了一点、什么都没讲透"的流水账。简介写得像立项书，AI 才知道**什么该详、什么该砍**。

### 怎么写：立项书四件套
一份合格的 `intro`，至少要回答这四个问题：

1. **主线是什么**——一句话主干（谁，因为什么，要干成什么，最大障碍是什么）。
2. **爽点/钩子在哪**——这片子靠什么让观众一集集追下去？是打脸、扮猪吃虎、悬念揭晓，还是情感拉扯？
3. **核心人物钩子**——主角的"反差"或"欲望"是什么（最抓人的一两个点）。
4. **基调与边界**——整体情绪（爽/虐/燃/暖），以及"不要走偏"的提醒（比如"不走纯升级流，重人物关系"）。

#### 范例对照

**范例 A：都市逆袭**

| | 内容 |
|---|---|
| Before（梗概式） | 讲述了一个落魄青年逆袭成为商业大佬的故事，期间经历了很多困难，最终走上人生巅峰。 |
| After（立项书式） | **主线**：被退婚羞辱的落魄继承人陈默，隐藏百亿身家回归，用三年时间夺回家族控制权并扳倒幕后黑手。**爽点**：每 2-3 集一次"身份揭穿/打脸"高潮，前期扮猪吃虎、后期正面碾压。**人物钩子**：陈默表面是被瞧不起的"废物女婿"，实则是商业天才，反差是最大看点；女主苏晴从轻视到动心，提供情感线。**基调**：爽快、节奏快、解气；不要拖沓的内心戏，每集结尾留一个反转钩子。 |

**范例 B：古风仙侠**

| | 内容 |
|---|---|
| Before | 一个少年踏上修仙之路，历经磨难，最终成为强者，守护苍生。 |
| After | **主线**：背负灭门血仇的少年凌霄，从凡人之躯一路问道登仙，目的不是变强而是揭开师门覆灭的真相。**爽点**：前期成长打脸，中期门派阴谋揭穿，后期宿命对决；情感上有一条"亦敌亦友"的双线。**人物钩子**：凌霄的执念（复仇 vs 苍生）构成内在撕裂；反派并非纯恶，有可信动机。**基调**：热血中带宿命悲剧感；避免无脑升级，每个境界突破都要有代价。 |

**范例 C：悬疑**

| | 内容 |
|---|---|
| Before | 一名刑警调查一起连环案件，最后抓到了凶手。 |
| After | **主线**：刑警林夏调查一桩跨越十年的连环失踪案，越查越发现案件与自己被遗忘的童年有关。**钩子**：靠"线索揭示 + 反转"驱动，每集抛一个新疑点、收一个旧疑点；真凶身份要藏到后三分之一。**人物钩子**：林夏既是调查者也是潜在受害者/知情者，身份的双重性是张力来源。**基调**：冷峻、压抑、智性；忌讳过早泄底，忌讳用巧合推动剧情。 |

### 常见错误
- **写成营销文案**："一部让你欲罢不能的神作！"——零信息量，AI 无从下手。
- **只写设定不写主线**：罗列了一堆世界观名词，却没说"谁要干成什么"。AI 会迷失在设定里。
- **主线与原著事件表打架**：简介说"重情感线"，但导入的章节全是打斗。骨架阶段 AI 吃的是事件表（见第02章），简介与事件不一致时，产出会精神分裂。
- **太长**：超过一屏的简介反而稀释重点。控制在 150–400 字，密度优先。

## 四、视觉手册 `artStyle` 与 导演手册 `directorManual`（选型决定成片质感）

这两个字段都不是自由文本，而是**文件夹名**——分别指向 `data/skills/art_skills/<artStyle>/` 和 `data/skills/story_skills/<directorManual>/` 下的提示词手册（见 `src/agents/productionAgent/index.ts` 的 `createArtSkills` / `useProductionSkills`）。

### 4.1 视觉手册 `artStyle`——"片子长什么样"

**它控制什么**：画面风格。它的影响面最广：
- 进**剧本 Agent** 项目语境（作"视觉参考"，让编剧知道这是漫剧还是真人剧，措辞会相应调整）；
- 进**制作 Agent** 的资产生成、分镜生成、视频提示词生成（`u.getArtPrompt(...)`），是成片视觉一致性的根。

**为什么重要**：画风一旦定错，后面所有图、所有镜头全跟着错，而且改一次=重做资产。它比模型选择更"伤筋动骨"。

**当前可选项**（`data/skills/art_skills/` 实际目录）：

| 文件夹 | 通俗叫法 |
|---|---|
| `2D_90s_japanese_anime` | 90 年代日系动画 |
| `2D_chinese_guofeng` | 2D 国风 |
| `2D_flat_design` | 2D 扁平插画 |
| `2D_mature_urban_romance` | 2D 都市成熟言情 |
| `3D_anime_render` | 3D 动画渲染 |
| `3D_chinese_traditional` | 3D 国风古典 |
| `3D_clay_stopmotion` | 3D 黏土定格 |
| `3D_guofeng_cyber` | 3D 国风赛博 |
| `realpeople_ancient_chinese` | 真人古装 |
| `realpeople_modern_city` | 真人现代都市 |
| `realpeople_urban_modern` | 真人都市现代 |

### 4.2 导演手册 `directorManual`——"片子怎么讲故事"

**它控制什么**：制作 Agent 的**叙事/调度风格**（运镜倾向、节奏、镜头语言习惯）。注意——**它不进剧本 Agent**（剧本 Agent 的项目语境里没有这个字段，已在源码确认）。它只在制作阶段，和 `artStyle` 一起被 `createArtSkills` / `useProductionSkills` 加载。

**为什么重要**：同一个剧本，配"悬疑惊悚"导演手册和配"家庭温情"导演手册，分镜的景别、节奏、镜头停留完全不同。它是"画风之外，决定观感"的第二根支柱。

**当前可选项**（`data/skills/story_skills/`）：

`Comedy_humor`（喜剧）、`Coming_of_age`（成长）、`Family_warmth`（家庭温情）、`Historical_epic`（历史史诗）、`Horror_supernatural`（恐怖灵异）、`Hot_blooded_action`（热血动作）、`Mystery_thriller`（悬疑惊悚）、`Psychological_drama`（心理剧）、`Scifi_post_apocalypse`（科幻末世）、`Sweet_romance_novel`（甜宠言情）、`Urban_workplace_drama`（都市职场）、`Xianxia_fantasy`（仙侠玄幻）。

### 4.3 题材 × 画风搭配建议表

选型原则一句话：**画风服务题材的视觉预期，导演手册服务题材的叙事预期，两者要同向。**

| 题材 | 推荐 `artStyle` | 推荐 `directorManual` | 说明 |
|---|---|---|---|
| 都市爽剧/逆袭 | `realpeople_modern_city` 或 `2D_mature_urban_romance` | `Urban_workplace_drama` | 真人增强代入感与"打脸"的爽 |
| 古风仙侠 | `2D_chinese_guofeng` / `3D_guofeng_cyber` | `Xianxia_fantasy` | 国风画面 + 仙侠叙事节奏 |
| 甜宠言情 | `2D_mature_urban_romance` | `Sweet_romance_novel` | 柔和画风 + 情感拉扯节奏 |
| 悬疑/犯罪 | `realpeople_modern_city` | `Mystery_thriller` | 真人冷峻质感 + 悬念叙事 |
| 恐怖灵异 | `2D_90s_japanese_anime` / `realpeople_modern_city` | `Horror_supernatural` | 氛围营造优先 |
| 热血战斗 | `2D_90s_japanese_anime` / `3D_anime_render` | `Hot_blooded_action` | 动作画面 + 燃向节奏 |
| 历史/年代 | `realpeople_ancient_chinese` / `3D_chinese_traditional` | `Historical_epic` | 厚重质感 + 史诗节奏 |
| 科幻末世 | `3D_guofeng_cyber` / `3D_anime_render` | `Scifi_post_apocalypse` | — |

**常见错误**：
- 画风与导演手册"拧着来"：选了 `3D_clay_stopmotion`（黏土定格）却配 `Horror_supernatural`，黏土的萌感会消解恐怖氛围。
- 把导演手册当画风选：以为选了 `Mystery_thriller` 画面就会变暗——不会，画面归 `artStyle` 管。
- 中途改 `artStyle`：已生成的资产图不会自动跟着变，需要重做。**画风请在出图前定死。**

## 五、视频画幅 `videoRatio`

### 它控制什么
出图与出片的尺寸（见 `batchGenerateImage.ts` / `generateVideo.ts` 里的 `aspectRatio`），同时进剧本 Agent 项目语境作"目标画幅"参考。默认 `16:9`。

### 怎么选

| 画幅 | 适用场景 |
|---|---|
| `9:16`（竖屏） | 抖音/快手/视频号短剧、漫剧、手机端为主的分发。**这是短剧/漫剧的主流，绝大多数情况选它。** |
| `16:9`（横屏） | 横屏长视频平台、电视端、电影感叙事、横屏切片 |

**常见错误**：做手机端短剧却用了默认的 `16:9`，成片左右大片留黑边，观感廉价。**做漫剧/短剧，先把它改成 `9:16`。** 画幅会影响构图，最好开工前就定。

## 六、图像/视频模型与质量：`imageModel` / `videoModel` / `imageQuality` / `mode`

这四个字段**只进制作 Agent，剧本阶段完全无关**。改它们不会改变剧本一个字。

- **`imageModel` / `videoModel`**：用哪个供应商的哪个模型出图/出片，格式为 `vendorId:modelName`（制作 Agent 会 `split` 出来用）。
- **`imageQuality`**：图像清晰度，取值 `1K` / `2K` / `4K`（见 `batchGenerateImage.ts` 的 `size`）。越高越清晰也越慢越贵。常规分镜 `2K` 足够；要做主视觉/封面再上 `4K`。
- **`mode`**：视频参考模式。制作 Agent 会尝试 `JSON.parse(mode)`，**如果解析出来是数组，就判定为"多参模式"（多图参考）**；否则按首尾帧等模式走（见 `productionAgent/index.ts` 的 `isRef` 逻辑与 `generateVideoPrompt.ts` 的模式路由）。这个字段决定了视频提示词用哪套模板，**应与你选的视频模型能力匹配**——选了支持多图参考的模型就用多参，反之用首尾帧。

**常见错误**：以为换个更强的图像模型能修好"剧情不对"——不能，剧情是剧本 Agent 的事，回去改 `intro`/`type`/对话。模型只决定"画得好不好看"，不决定"讲得对不对"。

## 七、想改 X，该回头调哪个字段？（速决表）

| 你的不满 | 病根字段 | 在哪个 Agent 生效 |
|---|---|---|
| 剧情主线跑偏、爽点不对 | `intro` | 剧本 Agent |
| 节奏不对、套路不像这个类型 | `type`（必要时配 `directorManual`） | 剧本 / 制作 |
| 画面风格不对、人物长得不对味 | `artStyle` | 制作 Agent（也影响剧本语境） |
| 运镜/镜头叙事感不对 | `directorManual` | 制作 Agent |
| 成片有黑边/比例不对 | `videoRatio` | 制作 Agent |
| 图糊/不够清晰 | `imageQuality` | 制作 Agent |
| 图/视频质量整体差 | `imageModel`/`videoModel` | 制作 Agent |
| 视频参考方式不对（多图/首尾帧） | `mode` | 制作 Agent |

> 黄金法则：**"讲什么"的问题往剧本字段找（name/type/intro），"长什么样、怎么拍"的问题往制作字段找（artStyle/directorManual/画幅/模型/质量/mode）。**

## 本章调优速查表

| 字段 | 一句话作用 | 进剧本 Agent | 进制作 Agent | 填写要点 | 头号坑 |
|---|---|---|---|---|---|
| `name` | 题材第一信号 | ✅ | — | 用真实书名 | 用编号/测试名 |
