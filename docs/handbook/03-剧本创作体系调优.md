# 第03章 · 剧本创作体系调优

## 本章导览

这一章是给你——拿着一部小说、想把它变成一部"卖得动"的漫剧的创作者——的剧本引擎说明书。

Toonflow 的剧本不是一个黑盒大模型一口气吐出来的。它是一条**三步串行流水线**，每一步背后都有一个独立的 Agent，每个 Agent 的"创作观念"都写在一个可编辑的技能 Markdown 文件里。改文件即时生效，不用重新打包。也就是说，**这些技能文件就是你的创作旋钮面板**。你想让剧本更爽、更虐、节奏更快、钩子更狠、质检更严，本质上都是在改这几段文字。

先把全局结构钉在脑子里：

```
项目初始化（你和决策层确认参数）
        ↓
阶段1  故事骨架   storySkeletonAgent      data/skills/script_execution_skeleton.md
        ↓        ↑ 监督层审核
阶段2  改编策略   adaptationStrategyAgent  data/skills/script_execution_adaptation.md
        ↓        ↑ 监督层审核
阶段3  逐集剧本   scriptAgent（循环 N 集）  data/skills/script_execution_script.md
```

外面还套着两层管理者：

| 层 | 文件 | 管什么 |
|---|---|---|
| 决策层 | `data/skills/script_agent_decision.md` | 流水线调度、和你对话、派发指令、把控阶段顺序 |
| 监督层 | `data/skills/script_agent_supervision.md` | 阶段1/2 产出物质检、打分 A/B/C/D、红线把关 |

各步看到的上下文不同，这决定了"问题会在哪里被埋下、会不会遗传"：

| 阶段 | 看得到的输入 |
|---|---|
| 故事骨架 | 事件表 + 项目语境（类型/简介/画风/画幅）+ 集数时长约束 |
| 改编策略 | 故事骨架 + 事件表 |
| 逐集剧本 | 故事骨架 + 改编策略 + **前一集已写剧本（承接）** + 当集对应原著正文 + 事件表 |

记住一句话贯穿全章：**越靠前的旋钮，影响越深、越难补救。** 骨架定生死，改编定调性，逐集只是把前两步的决定落成台词和画面。想要某种风格，优先去改骨架和改编，而不是逐集去硬掰。

下面逐文件拆解，每个旋钮我都会讲清楚：①控制什么 ②创作意义 ③怎么调 ④before/after ⑤常见错误。

## 一、决策层 `script_agent_decision.md`：流水线的"总导演台"

决策层不写一个字的剧本，它是唯一跟你对话的 Agent，负责把你的意图翻译成派发指令、按顺序推进流水线、把监督层的报告转述给你。它本身可调的"创作旋钮"不多，但它定义了**整条流水线的节流阀**，调错了会让前面所有努力打折。

### 旋钮 1.1 ｜ 项目参数表（第 26-36 行）——所有下游的唯一基准

- **控制什么**：集数、单集时长、原著范围、平台规格（画幅）、风格定位、付费策略。这六项会被原样拼进**每一条**派发给执行层和监督层的指令头部（第 51-63 行的「参数传递模板」）。
- **创作意义**：这是整部剧的"骨相"。监督层第 72 行写明"数值判断以【项目配置】为唯一基准"——也就是说，你填的集数和时长，直接决定了付费点落在第几集、骨架表必须有多少行、每集台词多少字。
- **怎么调**：你不改文件，你在对话里改。但要知道几个联动公式：
  - 台词字数 = `单集时长 × 150`（第 65 行，150 字/分钟语速）。
  - 付费点位置 = `⌈N×0.10⌉ / ⌈N×0.30⌉ / ⌈N×0.50⌉ / ⌈N×0.70⌉ / ⌈N×0.90⌉` 集（骨架文件第 136 行）。
- **before/after**：

  | | 设定 | 下游后果 |
  |---|---|---|
  | before | 100 集 × 2 分钟 | 付费点在第 10/30/50/70/90 集，每集约 300 字台词，节奏舒缓、可铺长线 |
  | after | 20 集 × 1.5 分钟（微短篇） | 付费点压到第 2/6/10/14/18 集，第 1 集要塞常规剧 3-4 集信息量（骨架第 118 行），节奏极限快 |

- **常见错误**：只给一半参数就想开工。决策层第 49 行规定"对未给出的参数逐一追问，不可使用默认值跳过"——但如果你含糊回答，它会照单全收。**集数和时长一定要想清楚再说**，中途改这两个值等于整个骨架重做。

### 旋钮 1.2 ｜ 派发指令 100 字上限（第 137-139 行）

- **控制什么**：决策层派给执行层/监督层的指令正文（不含配置头）严格 ≤100 字。
- **创作意义**：这是个"信任执行层"的设计——执行层的技能文件已经很全，决策层只传任务类型和关键差异点。**这意味着你在对话里对决策层提的个性化创作要求（"这集要特别虐""加一个反转"），只有压缩进这 100 字才会传到执行层。**
- **怎么调**：一般不动。如果你发现自己的特殊要求经常"传丢了"，可以把上限放宽到 150-200 字，或者干脆把高频要求写进对应执行层技能文件（一劳永逸，见下文各节）。
- **常见错误**：以为对决策层说一长段需求就会被忠实传达。实际上超出部分会被它自行裁剪概括，细节易失真。**反复要用的偏好，写进技能文件，别每次口头交代。**

### 旋钮 1.3 ｜ 阶段3 单次循环上限 5 集（第 127 行）

- **控制什么**：逐集剧本一轮最多生成 5 集，默认 3 集；超过会提醒"上下文超载"。
- **创作意义**：这是质量与吞吐的权衡。**集数越多，靠后那几集能"看到"的前文承接越被压缩，剧情走样风险越高**（承接机制见第三节）。
- **怎么调**：求稳每轮 3 集；赶进度可顶到 5 集，但建议在每轮之间人工抽查衔接。
- **常见错误**：硬要一次写 10 集图省事，结果第 8、9 集人物状态/称谓和前文对不上。

### 旋钮 1.4 ｜ 推荐分支（第 39-43 行）

- **控制什么**：当你说"帮我推荐配置"时，决策层会先问你想做的剧集形态（微短剧/短剧/长剧），再读事件表分析，给出推荐集数/时长/风格/付费策略。
- **创作意义**：拿不准节奏档位时的好起点。它会基于实际事件量给"推荐原因"。
- **怎么调**：无需改文件，直接在对话里触发"我不知道怎么配，帮我推荐"。
- **常见错误**：把推荐当定论。推荐只是基于事件数量的机械估算，**类型爽点的贴合度还得靠你在小说简介/类型字段里喂对（见第六节）**。

### 决策层的"红线"提醒（不是旋钮，但你要知道）

第 8、232-234 行反复强调：**subagent 失败时决策层绝不自己接管**。所以如果你看到"任务未完成"，那是真的没写成，不是偷偷降级写了个残次品——这时你要做的是重试或排查，而不是以为已经有产出。

## 二、监督层 `script_agent_supervision.md`：质检的"严苛程度"旋钮

监督层只审阶段1（骨架）和阶段2（改编策略）——**阶段3 逐集剧本不过监督层**（决策层第 125 行）。它的核心价值是：把"爆款短剧标准"变成一张可勾选的红线清单，逐项核对后打分。

**想要更严或更松的质检，就在这个文件里调。**

### 旋钮 2.1 ｜ 评分标准阈值（第 57-64 行）——质检宽严总闸

- **控制什么**：A/B/C/D 四档由"严重问题数 + 中等问题数"决定。

  | 评分 | 严重 | 中等 | 含义 |
  |---|---|---|---|
  | A 可直接用 | 0 | ≤2 | |
  | B 小修后可用 | 0 | ≤5 | |
  | C 需较大修改 | 1-2 | 不限 | |
  | D 建议重做 | ≥3 | 不限 | |

- **创作意义**：这张表直接决定你会多频繁地被打回重做。决策层第 183-188 行按评分给不同引导语（A→"是否进入下一阶段"，D→"建议重做"）。
- **怎么调**：
  - **想更严**（高标准产品）：把 A 的中等问题上限从 ≤2 收紧到 ≤1，把 C 的严重问题门槛从"1-2"改成"1"。这样小毛病也逼你修。
  - **想更松**（快速出片、自己后期再打磨）：把 B 的中等上限从 ≤5 放宽到 ≤8。
- **before/after**：

  ```
  before:  | A | 0 | ≤2 |
  after :  | A | 0 | ≤1 |   ← 几乎零容忍，骨架要打磨到很干净才放行
  ```

- **常见错误**：把阈值调得过松后忘了——结果烂骨架一路绿灯进了逐集，等发现时已经写了十几集。**调松阈值=把质检责任移交给你自己，要有觉悟。**

### 旋钮 2.2 ｜ 审核维度的"严重程度"标定（第 136-151 行 / 236-251 行）

- **控制什么**：每个审核项被标成"严重/中等/轻微"。由于评分只数"严重"和"中等"，**把一项从"中等"提到"严重"，等于给它加权**——它一旦不达标就可能直接把评分拖到 C/D。
- **创作意义**：这是"质检维度权重"的真正所在。你最在乎什么，就把它提级。
- **怎么调**（骨架审核表为例）：
  - 你做**强爽剧**，最怕"情绪布局"出问题 → 把"情绪布局"从`中等`改成`严重`，连续 3 集无爆点会被一票否决。
  - 你做**烧脑悬疑**，最怕反转空降 → "股价级反转登记"本就是`严重`，可再把"信息差标注"从`中等`提到`严重`。
  - 你想**放宽**对节奏框架的死板要求 → 把"节奏框架"从`轻微`保持或干脆移出维度表。
- **before/after**：

  ```
  before: | 情绪布局 | 全剧波浪上升…无连续3集同强度 | 中等 |
  after : | 情绪布局 | 全剧波浪上升…无连续3集同强度 | 严重 |
  ```
  改完后，"连续 3 集同强度"从扣中等分变成直接判严重，更容易触发 C/D 重做。

- **常见错误**：把太多项都提成"严重"。这样几乎每次都会判 D，流水线寸步难行。**一次只提级 1-2 个你这部剧最在意的维度。**

### 旋钮 2.3 ｜ 短剧通用红线（第 111-123 行）——一票否决项

- **控制什么**：9 条红线，违反任一条直接标"严重问题"。包括：连续 3 集无情绪爆点、多线并行、第 1 集无强冲突、出现"市长/县长"、大段旁白讲世界观、金手指同质化、无明确反转/反转空降、开篇踩三天坑、大三角只堆吵架。
- **创作意义**：这是 Toonflow 对"什么叫不及格短剧"的硬定义。
- **怎么调**：
  - **增加红线**：你的平台对某类内容零容忍（如绝不能出现现代货币），就往这张表加一条。
  - **删/松红线**：你做的是**长剧**而非短剧，"多线并行=严重"这条就该删（长剧允许多线）。这是最常见的需要松绑的一条。
- **常见错误**：做长剧/群像剧却没删"多线并行"和"≤4 人小传"的约束，结果监督层把你正常的双主线判成严重问题，反复打回。**剧集形态变了，红线也要跟着改。**

### 旋钮 2.4 ｜ 审核原则"可执行优先"（第 67-68 行）

- **控制什么**：第 2 条原则"标准是能不能用，不是完不完美"。
- **创作意义**：默认监督层是"务实派"，不挑文学性毛病。
- **怎么调**：追求精品可把这条改成"高标准优先，宁缺毋滥"，监督层会更挑剔。但通常不建议动——会拖慢整体节奏。

> **关于阶段3不审核**：逐集剧本没有监督层兜底。`script_execution_script.md` 第 385-404 行的「自查清单」是逐集剧本**唯一**的质量防线，且由执行层自己执行。所以如果你想给逐集也加"质检力度"，办法是去**强化那份自查清单**（见第三节末尾），而不是指望监督层。

## 三、故事骨架 `script_execution_skeleton.md`：决定"好不好卖"的地基

这是整条流水线里**最该花时间调**的文件。骨架不是把章节摊到集数上，而是"在结构层为好卖打地基"（第 41 行）。下面的旋钮按你最可能想调的顺序排列。

### 旋钮 3.1 ｜ 压缩比 ≤40%（第 32 行）

- **控制什么**：成片对原著的压缩程度上限。
- **创作意义**：决定"贴原著"还是"大刀阔斧"。40% 意味着最多砍掉六成原著。
- **怎么调**：忠实改编（粉丝向）→ 调高到 ≤60%，保留更多；爆款快剪向 → 维持 ≤40% 甚至更狠。
- **常见错误**：压缩比和集数/时长打架。比如 100 章小说硬塞 20 集 × 1.5 分钟，实际压缩远超 40%，骨架会被迫大删，监督层"章节全覆盖"（严重项）反而会报警。**先用集数×时长估算容量，再定压缩比。**

### 旋钮 3.2 ｜ 心理级爽点三选一 + 金手指原创性（第 73-83 行）——爽感的"频道选择器"

- **控制什么**：故事核必须锁定**一种**核心心理级爽点：
  - **优势/金手指**——主角独有能力，让观众"歪歪或崇拜"（战神、赘婿、神医）
  - **归属**——团结合作/家国情怀（帮派、大女主、女战神）
  - **秩序**——用逻辑还原真相（复仇、宫斗、悬疑、重生、寻亲）
- **创作意义**：这是你这部剧"爽在哪"的总开关。选错频道，再多反转也挠不到痒处。
- **怎么调**：在派发骨架指令时明确指定，或在小说简介里强暗示（见第六节）。
  - 想要**强复仇打脸** → 锁定"秩序"（逻辑推进还原真相 + 打脸）。
  - 想要**热血团魂** → 锁定"归属"。
  - 想要**无脑爽逆袭** → 锁定"优势/金手指"。
- **before/after**：

  ```
  before（频道混乱）: 故事核=主角重生复仇，顺便谈恋爱，还有个团队一起打拼
  after （锁定秩序）: 故事核=用记忆中的"未来情报"一步步揭穿仇人当年构陷的真相并反杀
  ```
  后者把爽点钉死在"秩序/逻辑还原"，所有反转都为它服务，不散。

- **常见错误**：贪心想三种爽点都要。文件明确"骨架须锁定核心 1 种"。**多频道=没频道。** 另外注意第 78 行：生理级爽点（性/暴力）易踩审核线，慎用。

### 旋钮 3.3 ｜ 矛盾四级阶梯（第 64-69 行）——"虐/张力"的强度档

- **控制什么**：大三角主矛盾要达到哪一级。爆款须达 3-4 级。
  1. 基本矛盾（太弱，平淡）
  2. 强化矛盾（强欲望+强阻碍+二选一困境）
  3. 高级矛盾（**两个好人因不同选择走向不同命运**）
  4. 矛盾升级（主角的行动招致更严重、不可回头的后果）
- **创作意义**：这是"戏的深度"旋钮。金句：最好的矛盾不是好人打坏人，而是两个好人因不同选择走向不同命运。
- **怎么调**：
  - 想要**纯爽不费脑** → 钉在 2 级（强化矛盾），反派就是纯坏人，主角碾压打脸。
  - 想要**有嚼劲、能虐到心里** → 推到 3 级，把反派写成"动机也成立的另一个好人"。
- **before/after**：

  ```
  before（2级）: 反派纯粹嫉妒主角，处处使绊子
  after （3级）: 男主为救垂危女儿抢水，反派抢的水是给奄奄一息的妻子——给谁都对
  ```

- **常见错误**：误把"堆吵架打斗"当矛盾。第 62 行：矛盾是内在静态的"想要而得不到"，冲突才是外在对抗。监督层第 91 行把"只堆吵架"列为严重问题。

### 旋钮 3.4 ｜ 付费点位置 + 5 大标准（第 134-170 行）——留存与变现的命门

- **控制什么**：付费卡点的**位置比例**（10/30/50/70/90%）、5 大标准、4 类写法、假付费点、卡点三步法。
- **创作意义**：付费点是平台变现的咽喉。卡点位置和强度直接决定付费转化。
- **怎么调**：
  - **想更早收割**（钱前置）：把首个卡点比例从 `≈10%` 改到 `≈8%`，配合第 132 行"付费冲动前置到前 3 集"。
  - **想加密钩子**：付费点之间多设"假付费点"（第 157 行可多次设置）。
  - **想改卡点风格**：4 类付费点写法（身份差/感情错位/人物命运巨变/环境剧变，第 159-164 行）按你的题材选主用类型。
- **before/after**（卡点三步法的威力，第 165-169 行）：

  ```
  before（错误：戛然而止吊胃口）: 主角正要反击 → 黑屏"欲知后事请付费"
  after （正确：先喂饱再卡）:
    ①先爽透: 投屏证据+全行业通报+反派跪地求饶
    ②拉高预期: "当年你们害我家人的账，我会一笔笔讨回来"
    ③卡核心钩子: 中年男人"你曝光的证据全被我拦下" → 定格女主脸色骤变
  ```

- **常见错误**：把高潮直接掐断当卡点（第 165 行明确这是错误写法）。观众没吃到甜头凭什么留？**必须先释放、再拉新预期、最后卡钩子。** 另外注意第 176 行：付费点偏差超过 ±2 集监督层会报问题，别随意挪。

### 旋钮 3.5 ｜ 类型节奏框架（第 172-189 行）——题材的"节奏模板"

- **控制什么**：5 种成熟题材的分集节奏骨架：甜宠、虐恋（追妻火葬场）、萌宝、战神、重生。
- **创作意义**：这是"按类型套节奏"的快捷模板，监督层第 88 行会核对分集是否与对应框架吻合。
- **怎么调**：
  - 你的题材不在 5 种里（如"宫斗""无限流"）→ **新增一行你自己的节奏框架**，否则监督层用最接近的框架硬套你会被判"节奏不符"（虽然只是轻微项）。
  - 想要**更快进入爽点** → 把战神类的"隐藏身份受辱（1%~30%）"压缩到"1%~15%"，让打脸提前。
- **before/after**：

  ```
  before（战神原版）: 隐藏身份受辱(1%~30%)→身份曝光打脸(31%~60%)…
  after （快节奏版）: 隐藏身份受辱(1%~15%)→身份曝光打脸(16%~50%)…  ← 第3集就开始打脸
  ```

- **常见错误**：题材选错框架。萌宝剧硬套战神节奏，前期没有"带娃逆袭"的温情铺垫，情绪基调就崩了。

### 旋钮 3.6 ｜ 全局情绪布局 + 类型情绪占比（第 191-208 行）——"爽/虐/甜"的配比表

- **控制什么**：全剧情绪曲线（波浪上升），以及各类型的情绪占比：
  - 甜宠：甜 60% + 微虐 30% + 惊喜 10%
  - 复仇：压抑 40% + 爽感 50% + 解气 10%
  - 重生逆袭：爽感 50% + 期待 30% + 温暖 20%
  - 家庭伦理：共情 40% + 委屈 30% + 和解 30%
- **创作意义**：这是最直接的"风格调味盘"。你想更虐就加压抑/委屈占比，想更爽就加爽感占比。
- **怎么调**：
  - **更虐的甜宠** → 改成 甜 45% + 微虐 45% + 惊喜 10%。
  - **更爽的复仇** → 改成 压抑 25% + 爽感 65% + 解气 10%（压抑段缩短，更快进入打脸）。
- **before/after**：

  ```
  before: 复仇类：压抑40% + 爽感50% + 解气10%
  after : 复仇类：压抑25% + 爽感65% + 解气10%   ← 少铺仇恨，多给打脸
  ```

- **常见错误**：把压抑/虐的占比调得太高（如 60%）。监督层第 87、148 行"不允许连续 3 集同一强度"会报警，而且观众容易被虐到弃剧。**虐是手段不是目的，虐完一定要给爽的反弹。**

### 旋钮 3.7 ｜ 股价级反转登记表（第 235-243、395-404 行）——烧脑/惊喜的引擎

- **控制什么**：全剧约 **3 个**大反转，骨架阶段 100% 定死，三式可选：预期误导 / 人设颠覆 / 动机置换。每个反转的预埋集必须早于揭晓集。
- **创作意义**：第 237 行——"股价级反转从根上打破观众刚看开头就猜到结尾的固有预判，决定一部剧能否爆款"。这是悬疑/烧脑剧的命根，也是任何剧"出乎意料"的来源。
- **怎么调**：
  - **烧脑悬疑** → 把数量从"≈3"上调到"≈4-5"，并主用"动机置换"和"预期误导"。
  - **简单爽剧** → 维持 3 个，主用"人设颠覆"（最易理解）。
- **before/after**（登记表填法）：

  ```
  | # | 反转类型 | 一句话描述 | 预埋集 | 揭晓集 | 兑现方式 |
  | 1 | 动机置换 | 观众以为女主熬药为救夫，真相是为复仇制毒摸破绽 | 第2,5,9集 | 第14集 | 揭晓时回放熬药镜头，旧线索全对上 |
  ```

- **常见错误**（也是红线，第 240、243 行）：
  - **人设颠覆/动机置换动了主角核心底色** → 观众失去代入当场弃剧。**只能用在配角。**
  - **空降硬凹反转**（线索对不上、画面造假骗人）→ 监督层第 120 行直接判严重。**全程不藏信息、反转后线索严丝合缝**才合格。

### 旋钮 3.8 ｜ 开篇即绝境 / 一卡三招（第 107-133 行）——前 10% 的生死线

- **控制什么**：开篇结构。"三集定生死、十集定全篇、卡点要卡住"；第 1 集 2 秒防划走、写清主角性格/困境/目标/动机四要素；避三天坑。
- **创作意义**：第 120 行——"前 10 集决定剧本生死，缺一就被毙"。前 10 集还是投放素材库（≈10 个可剪 30 秒爆点）。
- **怎么调**：微短篇要把"一卡三招"的节奏整体前压（第 118 行：卡点提前至 6-7 集，第 1 集承载 3-4 集信息量）。
- **before/after**：

  ```
  before（三天坑）: 真千金第一次被接回豪门，紧张自卑地打量别墅
  after （报款）  : 真千金一进门甩假千金一耳光，砸烂行李箱"这家有它没我"
  ```

- **常见错误**：开篇铺背景/讲世界观/写景（三天坑，第 127 行 + 监督层红线第 121 行）。**开篇唯一任务是 2 秒留人，故事后面再讲。**

### 旋钮 3.9 ｜ 人物小传 ≤4 人（第 37、84-105 行）

- **控制什么**：只为大三角核心角色立传，全剧 ≤4 人。
- **创作意义**：短剧单线型，人多则散。
- **怎么调**：做**长剧/群像**时这是必须放宽的约束——改成"≤6 人"或允许小三角换人立传。但短剧务必守住 4 人。
- **常见错误**：原著人物多就全立传，导致主线被稀释。**砍人是短剧改编的基本功。**

### 关于阶段3的"质检"补强

逐集剧本不过监督层，唯一防线是 `script_execution_script.md` 第 385-404 行的自查清单。**想给逐集加质检力度，就往这份清单里加勾选项**（如"本集必须出现一次掀桌子级别的动作情绪"），执行层会照单自检。

## 四、改编策略 `script_execution_adaptation.md`：定"调性"与"删什么"

骨架定了"讲什么结构"，改编策略定"用什么原则把原著揉成短剧"。它看骨架+事件表，产出 3-5 条核心改编原则 + 删除决策 + 世界观呈现策略。

### 旋钮 4.1 ｜ 8 大核心要点（第 34-45 行）——改编的总基准

- **控制什么**：强画面感、台词精简、节奏极致快、只沿主线、降低理解成本、情绪大于一切、开篇给足期待感、展示不要告诉。
- **创作意义**：这 8 条是改编一切决策的标尺，监督层第 98、291-300 行逐条核对。
- **怎么调**：通常不动这 8 条本身（它们是短剧通识），但你可以**调它们的相对优先级**。比如做"忠实文艺向"改编，可以把第 3 条"节奏极致快"和第 6 条"情绪大于一切"的措辞软化，给"逻辑完整性"让位。
- **常见错误**：想要"信息量大"却没意识到第 2 条"台词精简"和第 6 条"情绪大于一切"会主动砍掉你舍不得的设定交代。**短剧的默认价值观是情绪 > 逻辑，要逆这个默认就得改文件。**

### 旋钮 4.2 ｜ 情绪基调映射表（第 76-85 行）——和骨架联动的调性锁

- **控制什么**：4 类型的情绪基调占比（与骨架第 203-207 行同源）。第 85 行铁律：基调一旦确定不要中途大幅更改。
- **创作意义**：这是"风格一致性"的锁。监督层第 99、302-305 行把"甜宠突然全家惨死"列为严重偏离。
- **怎么调**：**如果你在骨架里改了情绪占比，这里一定要同步改**，否则两份文件打架，监督层"情绪基调一致"会报问题。
- **常见错误**：只改了骨架的情绪占比，忘了改编策略这张表 → 两阶段不一致。**情绪配比是跨两个文件的旋钮，要成对改。**

### 旋钮 4.3 ｜ 删减决策优先级（第 98-116 行）——"砍刀"的方向

- **控制什么**：优先删（拖沓铺垫/重复内容/载体不支持/弱支线）vs 优先保留（情绪点/关系拉扯/付费铺垫/信息差/打脸时刻）。
- **创作意义**：决定原著的哪部分活下来。
- **怎么调**：
  - 你特别看重某类内容（如"群像感"），就把它从默认"优先删的弱支线"里**显式移到优先保留**。
  - 想更狠地提速 → 在"优先删"里加一条"任何不含情绪点的过场戏一律蒙太奇压缩"。
- **before/after**：

  ```
  before（优先保留）: 情绪点/关系拉扯/付费铺垫/信息差/打脸时刻
  after （甜宠加保留）: …以上 + 所有主角二人独处的相处戏（CP 糖分来源）
  ```

- **常见错误**：删减和骨架对不上。监督层第 257、278 行做"删减决策一致性"交叉检查——骨架标"保留完整"的，策略不能标删除，否则判严重。

### 旋钮 4.4 ｜ 类型创新三方向 + 反洗稿（第 47-63 行）——"原创性"旋钮

- **控制什么**：三条死路（模仿/抄桥段/洗稿）+ 三大创新方向（元素创新/类型融合/情节创新）+ "市面 >10 次就别用"判定。
- **创作意义**：第 53 行——"同质化金手指 = 同质化剧本 = 卖不出去"。这是商业生死线。
- **怎么调**：想做差异化爆款 → 在指令或策略里明确要求引入"元素创新"（如年龄/性别/背景反转）。
- **before/after**：

  ```
  before: 青年战神隐藏身份在豪门受辱
  after （年龄反转）: 退休的老年战神隐藏身份，被晚辈轻视，一朝亮剑
  ```

- **常见错误**：照搬公共桥段（认亲胎记、三个巴掌"狼心狗肺/不知感恩/有眼无珠"）。监督层第 119、283 行直接判严重。

### 旋钮 4.5 ｜ 世界观呈现策略（第 199-204、307-310 行）——降低理解成本

- **控制什么**：设定的出场节奏、解释度（模糊/暗示/明确）、世界观锚点角色、观众视角对齐对象。
- **创作意义**：决定观众"看不看得懂"。短剧红线之一是禁止大段旁白灌输世界观（监督层第 118 行严重项）。
- **怎么调**：复杂设定（修仙/末世）→ 强调"渐进式呈现、每次只透露一个设定点、通过对话/OS/VO 带出"；简单都市 → 解释度调到"模糊/暗示"即可。
- **常见错误**：开篇用旁白把世界观一次性讲完。**改成让一个锚点角色在冲突里把设定"演"出来。**

### 旋钮 4.6 ｜ AI 短剧改编约束（第 142-146 行）——可生成性

- **控制什么**：画面优先、每集要有可视进展、规避 AI 跳脸/画面不连贯/重复场景。
- **创作意义**：Toonflow 是 AI 出片，这条决定剧本"做不做得出来"。
- **怎么调**：一般保持。如果你的画风/资产对某类场景特别不稳定，可在这里补一条"规避 X 类场景"。

## 五、逐集剧本 `script_execution_script.md`：把决定落成台词与画面

这一步循环 N 集，每次只写一集，看骨架+改编+前一集剧本+原著正文。它不过监督层，所以这里的旋钮直接决定成片质感。

### 旋钮 5.1 ｜ 单集篇幅 ≤1000 字 + 时长 ±10 秒（第 27-28 行 / 379-383 行）——节奏密度闸

- **控制什么**：场景正文（不含文件头和梗概）通常 ≤1000 字；时长按项目配置 ±10 秒，台词按 150 字/分钟。第 28 行：与时长推算冲突时**以"短、密、紧"为准**。
- **创作意义**：这是单集节奏密度的硬约束。1000 字逼着你砍废戏。
- **怎么调**：想要**更密更快** → 把 1000 字降到 800；想要**更舒展**（长剧单集）→ 提到 1500。
- **常见错误**：把单集时长设得很长（如 5 分钟）却没同步放宽 1000 字上限 → 执行层在"时长要 750 字台词"和"正文≤1000 字"之间纠结，默认偏向"短密紧"，结果时长不达标。**时长和字数上限要匹配着调。**

### 旋钮 5.2 ｜ 节奏 3-15-45（第 92-99 行）——秒级钩子密度

- **控制什么**：3 秒一个情绪冲击 / 15 秒一个剧情变化 / 45 秒一个强期待 + 结尾反转钩子。
- **创作意义**：对应平台只看的停留率/完播率/互动率。这是最硬的"秒级"节奏标尺。
- **怎么调**：想要**极限快节奏**（投流向）→ 改成 3-12-30，钩子更密；想要**喘息感** → 放到 5-20-60。
- **before/after**：

  ```
  before（3-15-45）: 3秒绑匪喊撕票→15秒妹妹喊"哥别给钱"→45秒限12点前50万→结尾反转
  after （3-12-30）: 同样三个点压缩进更短窗口，单位时间钩子更密，更适合 15-30 秒投流切片
  ```

- **常见错误**：把窗口拉得太长（如 10-40-90），短剧会显得"温吞"，完播率掉。

### 旋钮 5.3 ｜ 三大情绪要点密度（第 41-58 行）——每集情绪保底

- **控制什么**：每集 500-800 字须覆盖爆点/虐点/爽点**至少一个**（硬性）。
- **创作意义**：这是单集情绪密度的底线。
- **怎么调**：想要**更高情绪浓度** → 把"至少一个"改成"至少两个，且明确先后顺序"。配合爽点公式（装+打脸+震惊+收获，第 54 行）。
- **常见错误**：一次性宣泄完所有情感（第 52 行：小情绪累积成大情绪爆发）。情绪要省着用、留着憋大的。

### 旋钮 5.4 ｜ 单集情绪公式 1+1+1（第 127-131 行）

- **控制什么**：1 核心情绪 + 1 辅助情绪 + 1 结尾钩子；禁忌：单集核心情绪 ≤2、配角情绪不能盖过主角、上下集情绪须衔接。
- **创作意义**：防止单集情绪混乱。
- **怎么调**：基本不动；想要更纯粹的情绪体验可强调"单集仅 1 核心情绪"。
- **常见错误**：一集塞三四种核心情绪，观众跟不上。

### 旋钮 5.5 ｜ 单集钩子级反转三式 + ≤1 个（第 153-159 行）

- **控制什么**：道具伏笔反转/情绪回弹反转/镜头错位反转，单集尽量 ≤1 个。两准则：画面 100% 真实、不能连续使用同一招。
- **创作意义**：在骨架的 3 个大反转之外，给每集补"小钩子"，服务完播与付费。
- **怎么调**：烧脑剧可放宽到"每集 1 个，类型轮换"；纯爽剧保持"≤1 个，主用镜头错位"（最易上手）。
- **常见错误**：一集塞好几个反转（第 90 行：一集塞七八个反转=低情节密度）。**反转贵精不贵多。**

### 旋钮 5.6 ｜ 台词节制（第 182 行）——竖屏阅读速度

- **控制什么**：单句台词 ≤20 字，单角色单次台词尽量 ≤50 字。
- **创作意义**：竖屏观众阅读速度有限，长台词=划走。
- **怎么调**：横屏/长剧可适当放宽到单句 ≤30 字。
- **常见错误**：写成话剧式长对白。第 202 行明确"上百字交代闲聊会被一眼毙"。

### 旋钮 5.7 ｜ 展示不要告诉 + 画面感五招（第 171-194 行）——AI 可拍性

- **控制什么**：动作>台词、闭坑自爆家门式台词；△描述要写"人怎么干"，含景别/光影/动作；镜头术语要融进△描述，**禁止**写成"全景·缓推·约6秒"式技术括注（第 194、413 行）。
- **创作意义**：△描述直接喂给 AI 生成分镜，写法决定画面质量。
- **怎么调**：基本是 AI 出片的硬规范，不建议松。
- **常见错误**：写小说化描写（"窗外月光似乎也在为他流泪"，没法拍，第 200 行五大硬伤之一）；或在△里加镜头技术括注（第 413 行禁止项）。

### 旋钮 5.8 ｜ 禁止输出内容（第 406-417 行）——保持剧本纯净

- **控制什么**：不输出台词字数统计、版本标记、幕/节拍时间标注、镜头技术括注、自查清单、内部标尺（三大密度评级/3-15-45/投流素材点）。
- **创作意义**：这些是内部标尺，写进正文会污染喂给后续生产 Agent 的剧本。
- **怎么调**：不要动。这是和下游生产流程的接口约定。

## 六、用"小说简介/类型"字段配合技能文件，让骨架贴合你的爽点

骨架 Agent 看到的"项目语境"里就含**小说类型、简介、画风、画幅、章节数**。这些字段是你在不改技能文件的前提下，**最低成本地引导骨架方向**的手段。技能文件定义"招式库"，项目字段告诉 Agent"这次用哪套招"。

实战要点：

1. **类型字段 = 节奏框架 + 情绪占比的选择器**。你填"复仇"，骨架会自动对齐第 188 行重生类/复仇情绪布局；填"甜宠"则走甜 60% 的配比。**类型填错，节奏和情绪全偏。**

2. **简介要写"爽点和钩子"，不要写"剧情流水账"**。骨架第 16 行第一步就是"核心吸引力判断"。简介里直接点明你要的心理级爽点和金手指会极大提升贴合度。

   ```
   弱简介: 讲述林某重生回到三年前，经历种种最终成功的故事。
   强简介: 复仇爽剧。女主带着前世记忆重生，用"已知的未来"逐个揭穿
           当年构陷她全家的三个仇人。核心爽点=秩序（逻辑还原真相+打脸）；
           金手指=每天只能预知一件未来小事（有约束）；情绪偏爽，少铺压抑。
   ```
   强简介等于把"心理级爽点、金手指及约束、情绪倾向"三件最关键的事提前喂给了骨架，省得它自己猜偏。

3. **画风/画幅字段影响改编与逐集**。画幅（竖屏/横屏）决定逐集第 354-358 行的构图规范（竖屏要人物居中、禁横向全景）；画风影响 AI 可生成性的判断。

4. **字段与技能文件要"同向"**。如果你既在简介里说"强爽少虐"，又把骨架情绪占比改成"压抑 60%"，两者打架。**先想清楚风格，再让字段和文件同向发力。**

## 七、逐集重写的注意事项：承接会遗传，问题也会遗传

逐集剧本是**循环**生成的，每写一集，执行层会通过 `get_script_content` 读**上一集**剧本来衔接剧情和角色状态（第 15 行）。这带来一个关键特性：

> **前文是"承接基准"，前文的问题会原样遗传给后文。**

具体要警惕：

1. **承接窗口只有"最后一集"**。第 30、233 行明确 `get_script_content` 只允许取最后一集。也就是说，第 5 集只直接看到第 4 集，看不到第 1 集的细节——**跨集的称谓/状态一致性，主要靠骨架兜底，而非逐集互看**。所以骨架里人物小传的"说话风格/标志物"要写准，它是防漂移的锚（第 101-103 行）。

2. **重写中间某一集，会和它的下一集断裂**。如果你回头重写第 3 集改了结局钩子，第 4 集是基于"旧第 3 集"写的，承接就对不上了。**重写一集后，它后面的集最好也跟着重写**，否则衔接处会出现矛盾。

3. **问题会沿着循环放大**。第 2 集主角性格写歪了，第 3 集承接第 2 集会把歪的延续下去，越往后越偏。**逐集生成时务必"边写边抽查"前几集，早发现早重写**，别等 20 集写完再回头救。

4. **单轮集数越多，承接越可能失真**。结合决策层旋钮 1.3：每轮 3 集、轮间抽查，是性价比最高的做法。

5. **重写不会自动重跑骨架/改编**。逐集重写只动那一集，骨架和改编策略不变。如果问题根源在骨架（如爽点频道选错），**重写多少集都没用，要回到阶段1**。

## 八、实战对照表："我想要 X 风格 → 这样改这些文件"

下面三套是最常见的风格诉求。每套都标出"改哪个文件的哪段、改成什么"。改完记得让监督层维度也同向调整（旋钮 2.2）。

### 风格 A：强复仇打脸（爽感拉满、压抑要短、打脸要狠）

| 文件 | 改哪段 | 怎么改 |
|---|---|---|
| 骨架 `skeleton.md` | 心理级爽点（旋钮3.2） | 锁定"**秩序**"（逻辑还原真相+打脸） |
| 骨架 | 情绪占比（旋钮3.6，第205行） | 复仇类改成 **压抑25%+爽感65%+解气10%**（缩短铺垫） |
| 骨架 | 矛盾阶梯（旋钮3.3） | 钉在 2 级即可（反派可纯坏），重点在打脸爽感 |
| 骨架 | 付费点（旋钮3.4） | 主用"人物命运巨变"写法，卡点三步法把"反派跪地求饶"写足 |
| 改编 `adaptation.md` | 删减保留（旋钮4.3） | 把"打脸时刻/身份反差"提到最高保留优先级 |
| 逐集 `script.md` | 三大情绪要点（旋钮5.3） | 强制套用模板1"打压-反击"（第221行），爽点用足"装+打脸+震惊+收获" |
| 监督层 `supervision.md` | 维度权重（旋钮2.2） | 把"情绪布局"提到`严重`，防止压抑段拖太长 |

### 风格 B：甜宠（糖要多、虐要轻、CP 感要强）

| 文件 | 改哪段 | 怎么改 |
|---|---|---|
| 骨架 | 类型节奏框架（旋钮3.5，第177行） | 用甜宠框架，可把"误会拉扯"段缩短，更快进撒糖 |
| 骨架 | 情绪占比（旋钮3.6） | 想更甜：**甜70%+微虐20%+惊喜10%**；想更虐恋：甜45%+微虐45% |
| 骨架 | 矛盾阶梯（旋钮3.3） | 用"误会/秘密"作软矛盾，别上升到你死我活 |
| 改编 | 情绪基调表（旋钮4.2，第80行） | **同步**改成与骨架一致的甜宠占比（成对改！） |
| 改编 | 删减保留（旋钮4.3） | 显式保留"主角二人独处相处戏"作为糖分来源 |
| 逐集 | 情绪模板/CP（第224、205行） | 主用模板2"误会-解开"甜虐布局；强化 CP 感技巧（性格互补反差萌） |
| 逐集 | 单集情绪公式（旋钮5.4） | 核心情绪锁"微甜"，辅助情绪用"女配吃醋"类小冲突 |

### 风格 C：烧脑悬疑（反转密、信息差精、逻辑严）

| 文件 | 改哪段 | 怎么改 |
|---|---|---|
| 骨架 | 心理级爽点（旋钮3.2） | 锁定"**秩序**"（逻辑推进还原真相） |
| 骨架 | 股价级反转（旋钮3.7，第243行） | 数量从"≈3"上调到"**≈4-5**"，主用"预期误导+动机置换" |
| 骨架 | 信息差设计（第216行） | 在更多关键集标注信息差类型，主用"上帝型/焦急型" |
| 骨架 | 矛盾阶梯（旋钮3.3） | 推到 3 级（两个好人不同选择），增加道德灰度 |
| 改编 | 世界观呈现（旋钮4.5） | 解释度调到"暗示"，靠观众推理，信息渐进释放 |
| 改编 | 反转来源（第136行） | 详写每个反转"从原著如何重构"，确保线索严丝合缝 |

> **跨风格通则**：无论哪种风格，**改情绪占比一定要骨架(3.6)和改编(4.2)成对改**；**改了爽点频道(3.2)就得回到阶段1重做骨架**，逐集修改救不了根；**监督层维度提级一次别超过 1-2 项**，否则永远判 D。
