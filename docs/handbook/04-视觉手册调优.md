# 第04章 · 视觉手册调优

> 把一套画风从「能出图」调到「就是我要的那个味儿」。

---

## 本章导览

在 Toonflow 里，「视觉手册」就是 `data/skills/art_skills/<风格>/` 这个目录——内置约 11 套风格（`2D_90s_japanese_anime`、`2D_chinese_guofeng`、`2D_flat_design`、`2D_mature_urban_romance`、`3D_anime_render`、`3D_chinese_traditional`、`3D_clay_stopmotion`、`3D_guofeng_cyber`、`realpeople_ancient_chinese`、`realpeople_modern_city`、`realpeople_urban_modern`）。每一套都是一摞 Markdown，**不是代码**，改完即时生效、不用重新构建。这意味着：作为美术指导，你不需要会写一行 TypeScript，就能把整套片子的调性按住。

但「能改」不等于「会改」。我见过太多人一上来就把色值乱调、把硬约束删光，结果脸崩、风格漂、四视图被裁，最后回来怪 AI 不行。这一章我会带你**逐个文件类型**拆解：每个文件控制什么、改了有什么创作意义、怎么调才稳、改前改后长什么样、哪些坑千万别踩。

先看清楚这套手册是怎么被系统吃进去的，否则你改了半天可能根本没生效：

| 文件 | 谁在用 | 怎么用 |
|---|---|---|
| `prefix.md` | 资产出图（`src/utils/getArtPrompt.ts`） | **每一张资产图**都会先拼 `prefix.md`，再拼对应的 `art_prompt/<类型>.md` |
| `art_prompt/art_character.md` 等 7 个文件 | 资产出图 | 按资产类型选其一，拼在 `prefix.md` 之后送进出图模型 |
| `driector_skills/` 三个文件 | 制作 Agent（`src/agents/productionAgent`） | Agent 通过 `activate_skill` 工具**按需加载**，用于导演级规划与分镜 |
| `README.md` | 人看的 | 风格说明，不进 prompt，但建议同步更新 |

**关键机制一**：`getArtPrompt(styleName, source, fileName)` 的逻辑是「`prefix.md` 内容 + 目标文件内容」拼接（见 `src/utils/getArtPrompt.ts` 第 29–30 行）。所以 `prefix.md` 是**全局基因**，改它会影响这套风格下的**所有**资产；而 `art_prompt/` 下的文件只影响对应资产类型。这是「调全局」还是「调局部」的分水岭。

**关键机制二**：`driector_skills/` 里的三个文件靠 frontmatter 的 `name` 字段被 `activate_skill` 精确匹配（`src/utils/agent/skillsTools.ts` 用 `z.enum(skillNames)` 做枚举校验）。也就是说，**`name` 字段不能乱改、不能重名**，否则 Agent 调不到这个技能、会直接报错。这是新建风格时最容易翻车的地方，本章末尾会专门讲。

下面进入正题。本章以 `2D_90s_japanese_anime`（90 年代日式动画，2D 手绘）为主线，穿插对比 `realpeople_modern_city`（真人都市实拍），让你看清同一个「旋钮」在 2D 与真人两种媒介下的不同表达。

---

## 一、`prefix.md` —— 全局基因（牵一发动全身）

`prefix.md` 是整套风格的「宪法」。它定义了**色板、情感配色映射、色温/饱和度、全局硬约束**。因为它会被拼到每一张资产图前面，所以**改它的杠杆最大、风险也最大**。先理解它的结构，再动手。

### 1.1 色板（C1–C10 含 hex）

> **控制什么**：整套风格的「调色盘上限」——所有资产默认从这十个色里取色。
> **创作意义**：它决定了片子的「色彩人格」。暖怀旧、冷克制、高级灰还是糖水甜，第一眼就由这十个 hex 锁定。

以 `2D_90s_japanese_anime/prefix.md` 为例，它的色板是这样的：

| 序号 | 色名 | 色值 | 用途 |
|---|---|---|---|
| C1 | 暖黄 | `#F5E6D0` | 肤色基底、暖光、夕阳 |
| C2 | 樱花粉 | `#F4D5D5` | 脸颊红晕、春日、浪漫 |
| C3 | 天空蓝 | `#87AEC9` | 天空、服装、冷调点缀 |
| C4 | 深棕发 | `#4A3728` | 发色、眼瞳 |
| C7 | 琥珀暖 | `#C9A96E` | 黄昏、灯光、温暖感 |
| …… | …… | …… | （C5/C6/C8/C9/C10 略） |

注意它顶部明确写了：「**全局色彩盘（风格基线，不是硬锁）**」，并划分了三个层级 L1 硬约束 / L2 软约束 / L3 例外机制。其中 **L1 硬约束色**只锁了两项：肤色基准（C1）、发色/瞳色基准（C4）。这是它故意留的口子——色板不是用来捆死你的，而是统一审美的「默认优先」。

#### 怎么调 hex 调整整体调性

改 hex 的核心原则是：**沿同一色相轻移，不要跨色相乱跳**。`prefix.md` 自己给了容差表（色相 ±8°、饱和度 ±10%、明度 ±12%），你的大改动应该落在「重设基准」而不是「在容差内抖」。

**Before / After 范例：把肤色基底调暖**

- **Before**：`| C1 | 暖黄 | #F5E6D0 | 肤色基底、暖光、夕阳 |`
- **After**：`| C1 | 蜜橘暖 | #F7DEB8 | 肤色基底、暖光、夕阳 |`

`#F5E6D0`→`#F7DEB8` 是把肤色基底往「橘」推了一档（R 略升、B 明显降），整片人物会更显健康、夕阳/暖光场景更浓。因为 C1 同时是 L1 硬约束的「肤色基准」，这一改会**同步影响所有人物的肤色锚点和所有暖光场景**——这就是全局杠杆的威力。如果你只想改某场戏的肤色，**别动这里**，去分镜阶段加局部色温词。

**实战提醒**：改完一个色，去「情绪色盘」和「硬约束色」两张表里搜一遍这个色号（如 C1），确认引用它的地方语义还成立。色板和下面的情感映射是联动的，单改 hex 不改映射，容易出现「色对了但情绪叙事错位」。

#### 常见错误

| 错误 | 后果 | 正确做法 |
|---|---|---|
| 把 C1–C10 全部替换成一套全新色 | 风格基因被掀翻，等于新建风格却没改 README/约束 | 大改请走「新建自定义画风」流程（见第六节） |
| 引入高饱和荧光色（如 `#00FF88`） | 直接撞 `prefix.md` 的严禁项 X3 | 荧光/霓虹是这套风格的红线，想要赛博去用 `3D_guofeng_cyber` |
| 只改色名不改 hex | prompt 里只用色名时看似变了，实际渲染没变 | 出图模型吃的是 hex 和语义词，hex 才是真旋钮 |

### 1.2 情感配色映射（情绪色盘）

> **控制什么**：把「剧情情绪」翻译成「主色 + 辅色 + 光效」。
> **创作意义**：这是「色彩叙事」的核心表。它保证「心动的戏是暖粉的、离别的戏是降饱和拉冷暖差的」，让全片的情绪和色彩对齐。

`2D_90s_japanese_anime` 的情绪色盘（节选）：

| 情绪场景 | 主色 | 辅色 | 光效与对比建议 | 画面关键词 |
|---|---|---|---|---|
| 心动瞬间 | C2 樱花粉 | C1 暖黄 + C10 暖橙 | 中近景提暖，肤色微红 | 羞涩、靠近感、暧昧 |
| 离别感伤 | C5 高级灰 | C3 天空蓝 + C1 暖黄 | 降饱和，拉大冷暖反差 | 距离感、克制、静压 |
| 重逢释怀 | C1 暖黄 | C7 琥珀暖 + C2 樱花粉 | 先冷后暖，面部暖光渐进 | 回温、释然、治愈 |

它还配了一套使用规则 E1–E5，其中两条最该记住：**E2 单镜头主色不超过 2 个**（避免颜色叙事失焦）、**E3 情绪切换时优先调光比与色温，再调饱和度**。

#### 怎么新增/修改情感配色

**新增一个情绪**——比如剧本里有大量「暗恋嫉妒」的戏，内置表没有，你可以加一行：

```
| 暗恋嫉妒 | C6 淡紫 | C5 高级灰 + C3 天空蓝 | 冷调压暗，局部暖点保留一丝不甘 | 隐忍、酸涩、克制 |
```

要点：**主色必须从已有 C1–C10 里选**（保证不破坏色板基因），辅色给两个，光效一栏要写「怎么打光」而不是「什么情绪」——这一栏会被导演阶段直接读走。

**修改已有情绪**——比如你觉得「黄昏浪漫」太腻，想更克制：

- **Before**：`| 黄昏浪漫 | C7 琥珀暖 | C10 暖橙 + C2 樱花粉 | 逆光暖调，轮廓光 | 怀旧、浪漫、情感 |`
- **After**：`| 黄昏浪漫 | C7 琥珀暖 | C3 天空蓝 + C5 高级灰 | 逆光暖调但天空压冷，轮廓光收敛 | 怀旧、距离、隐隐惆怅 |`

辅色从「暖橙+粉」换成「蓝+灰」，同样是黄昏，情绪从「甜」变成「怅」。这就是辅色的威力：主色定基调，辅色定情绪的细微走向。

#### 常见错误

- 一个情绪堆 4 个主色 → 违反 E2，画面花、情绪散。**最多 2 个主色**。
- 新增情绪用了色板外的颜色 → 全片色彩基因被局部污染。**主色锁在 C1–C10**。
- 只改情绪色盘、忘了 `driector_skills/director_planning_style.md` 里的「光影方案 A–F」——两边是配套的，光影方案也得同步调（见第五节）。

### 1.3 色温与饱和度

> **控制什么**：全片的「冷暖」与「浓淡」基准线。
> **创作意义**：色温是风格最底层的「体温」。同样的画面，4800K 是怀旧老动画，6500K 是现代清爽剧。

`2D_90s_japanese_anime` 的色温约束：

| 参数 | 值 | 说明 |
|---|---|---|
| 整体色温 | 偏暖 4800-5200K（推荐） | 温暖怀旧主基调 |
| 肤色色温 | 微暖 5000-5400K（推荐） | 暖黄但有生命感 |
| 饱和度 | 中低 50-70%（建议区间） | 90 年代动画高级色调 |

**改色温的效果（Before/After）**：

- **Before**：`整体色温 | 偏暖 4800-5200K`
- **After**：`整体色温 | 中性微冷 5600-6000K`

把基准从 4800K 推到 5800K，整片立刻「现代化」——怀旧滤镜褪去、空气变清透。但**警告**：这套风格的严禁项 X1 明文禁止「现代日系动画/新海诚后期/MAPPA 风格」。把色温拉冷拉清透，等于在往 X1 撞，画面会变成「新海诚味」。**如果你真要现代清新感，应该换风格而不是硬改 90s 这套的色温**。这正好说明：色温不是孤立旋钮，它和整套风格的「严禁项」互相咬合。

**对比真人风格**：`realpeople_modern_city/prefix.md` 完全没有「整体色温一个值」这种写法，它用的是**一整张「城市光源色谱」**——晨光 3500-4500K、金色时刻 2800-3500K、高压钠灯 2000-2200K、便利店荧光灯 4000-6500K……每个光源一个色温。因为真人实拍的逻辑是「光从哪个真实光源来」，而不是「全片统一一个色温」。所以在真人风格里调色温，你调的是**光源选择**，不是一个全局数字。

### 1.4 全局硬约束 R / X

> **控制什么**：这套风格「必须有什么（R）」和「绝不能有什么（X）」。
> **创作意义**：这是风格的「免疫系统」。它防止 AI 在自由发挥时漂出风格边界。

`2D_90s_japanese_anime/prefix.md` 的必守 R1–R5、严禁 X1–X6，例如：

- R1 必须包含「90 年代日式动画风格」锚定词
- R2 必须声明「手绘质感 + 平涂上色」
- X2 严禁「3D 渲染/CG 动画/数字绘画」
- X3 严禁「高饱和荧光色/霓虹色」

`realpeople_modern_city/prefix.md` 的硬约束更狠、分了三组（P1–P8 影像真实性、Q1–Q5 摄影品质、X1–X10 严禁），还多了一张独有的「**风格越界判定**」表——比如「人物皮肤像硅胶 = 已越界，立即停用『光滑/完美肌肤』类词汇」。这张表是真人风格的精华，本质是把「AI 出错的征兆」前置成可检查的清单。

#### 哪些 R/X 能动、哪些动不得

| 类别 | 例子 | 能否删/改 |
|---|---|---|
| 媒介锚定类 | 2D 的 R1/R2/X2；真人的 P1/X1/X2 | **绝对不能删**。删了媒介锚，AI 会在 2D 和 3D 之间乱跳，整片风格崩 |
| 质感锚定类 | 2D 的 R3/R4「手绘线条」；真人的 P3「保留毛孔」 | 不建议删。可微调措辞，但要保留语义 |
| 题材边界类 | 真人的 X3「严禁古风古装」 | **可按你的项目改**。如果你要拍古装真人，本就该用 `realpeople_ancient_chinese` 而不是改这里 |
| 内容安全类 | X9「严禁暴力血腥/暴露」 | 不要删 |

**Before/After 范例：放宽题材边界**（仅当确有需要）：

- 假设你在 `realpeople_modern_city` 上做一个「都市里偶尔闪回民国」的项目，X3 严禁民国会挡住闪回。正确做法**不是删 X3**，而是新建一个派生风格目录、在派生版里把 X3 改成「当代都市为主，民国仅限明确标注的闪回段落」。保留原风格干净，是为了别的项目还能用。

#### 常见错误

- 把 X 严禁项全删了图省事 → AI 失去边界，10 张里有 3 张漂风格。**X 是省钱的（少废图），不是碍事的**。
- 加 R 加太多太细（比如把每个配饰都写成必守）→ prompt 过长、模型抓不住重点、互相打架。**R 只放真正不可妥协的，5–8 条为宜**。

## 二、`art_prompt/art_character.md` —— 角色基础形象

> **控制什么**：角色的「素颜底模」——五官逻辑、肤感、体型/头身比、基础发型、基础着装、四视图设定图规范。
> **创作意义**：这是所有角色资产的**地基**。底模定不准，后面换装、分镜全跟着歪。

### 2.1 这个文件的核心设计

`2D_90s_japanese_anime/art_character.md` 有个很聪明的取舍：**五官不写死参数**。它第二节明说「不再固定五官特征参数，由角色描述（性别/年龄/性格/气质）驱动 AI 自由生成五官，保证人物间外观差异化」。它写死的是**风格底色**（手绘平涂、流畅线条、柔和暖色）和**比例规范**（女性 6–6.5 头身、男性 6.5–7.5 头身）。

`realpeople_modern_city/art_character.md` 走得更远：它根本不按「脸型/眼型」解剖分类，而是按「**摄影机如何阅读一张脸**」给了女性 5 种 + 男性 5 种「面孔类型」（清冷克制型 / 温柔治愈型 / 都市干练型 / 青春元气型 / 市井烟火型）。每种都带「摄影关系」（适合多少 mm 镜头、什么光）。这是真人风格的精髓——**选角思维**而非建模思维。

### 2.2 怎么调让角色更贴某种气质

**目标：让女主整体更「清冷高级」**

在 `2D` 这套里，气质是「从角色描述提炼关键词写进 prompt」的（第二节通用要求）。你不改文件也能在角色设定里写「清冷」，但如果你想让**这套风格下的女性默认更清冷**，可以调「体型/肤感」这两节：

- **Before（肤色）**：`| 肤色 | 冷白皮、暖调米白 | 冷白皮、暖调米白 |`
- **After**：`| 肤色 | 冷白皮、偏青冷调 | 冷白皮、冷调通透、少血色 |`

- **Before（体态）**：`| 体态 | 优雅/温柔、体态轻盈 | 体态轻盈、身姿优雅 |`
- **After**：`| 体态 | 清冷疏离、体态挺拔 | 身姿挺拔、神态淡漠克制 |`

在真人那套里更直接：你在角色设定阶段直接点名「类型 A 清冷克制型」，prompt 就会带上「窄内双/单眼皮、偏长眼裂、克制的注视、疏离而不冷漠」「微冷白肤色、T 区轻微出油」「适合 50mm 近距离凝视、侧面光勾勒骨感」。**真人风格调气质 = 选对面孔类型 + 选对镜头/光**，几乎不用改文件。

### 2.3 四视图规范——哪些硬约束不能删

四视图设定图是底模的「身份证」，它的约束是为了**后续一致性**服务的，删了会连锁出问题：

| 约束 | 删了会怎样 | 结论 |
|---|---|---|
| R6 全身立像从头到脚完整、严禁裁切（X6） | 出图经常裁掉脚或头顶，做动画时角色「没脚」 | **绝对保留** |
| R8 人像特写从头顶到锁骨完整（X7） | 特写裁额头，后续换装贴脸基准丢失 | **绝对保留** |
| R5/R7 四视图一致性 + 头身比换算 | 四个视图像四个人，或正面侧面身高不一 | **绝对保留** |
| R4 暖调米白背景 `#F8F4E8` | 背景花了，img2img 抠不干净 | 颜色可改，但「纯净单色背景」语义要保留 |
| R3 无发饰无配饰 | 底模带了发饰，换装阶段没法干净叠加 | 保留——发饰是衍生层的事 |

真人那套对应的是中灰无缝背景纸 `#B0B0B0`、棚拍柔光、「同一人物连续棚拍记录」，逻辑一样：底模要**干净、完整、可复用**。

### 2.4 提示词模板长什么样

`art_character.md` 第八节直接给了可填空的模板，系统出图时把角色信息填进 `{}`：

```
{性别}角色四视图设定图，90s anime style，复古日系动画风格，手绘平涂上色，柔和暖色调，精细流畅线条，电影感光影，
character design sheet，character turnaround，
{角色描述对应的五官特征 - 由角色描述自然推导}，{整体气质}，素颜无妆，
{身高描述}，{头身比，如：6.5 heads tall proportion}，{体态描述}，
...
全身立像从头顶到脚底完整展示，full body head to toe，不裁切头顶和脚部，
图中不要有任何文字
```

**你能调的就是这个模板的固定部分**（风格锚词、约束句），`{}` 部分由角色数据动态填充——所以**不要把 `{}` 占位符删掉或写死**，否则角色差异化就没了。

#### 常见错误

- 把头身比约束删了（X8 忽略身高头身比）→ 同一角色不同图忽高忽矮。
- 给底模加了浓妆/发饰/复杂花纹 → 违反 R1 素颜、R3 无配饰，后续衍生层叠加时打架（衍生文件 X1 明确禁止复杂花纹干扰）。
- 在 2D 风格里写真人化的「毛孔/细纹」→ 撞 X2「数字化锐利边缘」，画面会变得既不像手绘也不像真人。

## 三、`art_prompt/art_character_derivative.md` —— 换装/变身/妆造

> **控制什么**：在底模之上**叠加**妆容、发型、服饰、鞋履、配饰（L0–L6 分层），以 img2img 方式生成。
> **创作意义**：一个角色一套底模，无数套造型。这个文件保证「换装不换脸」。

### 3.1 L0–L6 分层是怎么回事

`2D_90s_japanese_anime/art_character_derivative.md` 的分层：

| 层级 | 内容 |
|---|---|
| L0 | 底模（不修改） |
| L1 | 妆容（系统先分析线索再决策强度：基础妆/轻妆/正式妆/病弱妆） |
| L2 | 发型造型 + 发饰 |
| L3 | 中衣/内搭 |
| L4 | 外衣/主服 |
| L5 | 鞋履 |
| L6 | 配饰 |

分层的意义是**可独立替换**——换一件外衣只动 L4，不碰脸（L0）和妆（L1）。它的提示词模板也是分层写的：

```
以角色基础形象图为底图，img2img叠加服化妆造，
保持基础形象面容不变，{整体气质}，
【L1·妆容】{基础妆/轻妆/正式妆}；使用 {妆容风格}，{眉妆}，{眼妆}，{唇妆}，
【L2·发型】{造型类型}，{发饰描述}，
【L3+L4·服饰】{主色}{款式}，{材质}，{装饰工艺}，
【L5·鞋履】{鞋款}，{鞋面材质}，
【L6·配饰】{头饰}，{耳饰}，{项饰}，
...
四视图一致性，面容细腻渲染
图中不要有任何文字
```

### 3.2 衍生 prompt 怎么写、怎么避免漂脸

「漂脸」（叠加后五官变了）是衍生环节的头号灾难。这个文件用**四道锁**防它，写衍生 prompt 时务必全部带上：

1. **img2img 而非 text2img**——模板第一句「以角色基础形象图为底图，img2img 叠加」。这是物理基础，丢了它就是重新生成一个人。
2. **显式声明面容不变**——R1「叠加后面容必须与底模一致」、X1「严禁面容偏移」。模板里写死「保持基础形象面容不变」。
3. **姿态锁定**——R11/X10「保持底模自然站立姿态」。一旦让它换姿势，脸大概率跟着变。
4. **范畴隔离**——R9/X7「禁止包含场景描述」、R10/X9「禁止道具交互」。衍生资产只管「穿在身上的」，场景和手持物是别的资产的事。混进来会挤占模型注意力、诱发变形。

**Before / After 范例：写一套「正式宴会装」衍生**

- **Bad（会漂脸）**：
  > 生成女主穿红色礼服站在宴会厅，手举香槟，灯光璀璨，妆容精致……
  >
  > 问题：text2img 描述、带了场景（宴会厅）、带了道具（香槟）、改了姿势（手举）——四道锁全破，脸必崩。

- **Good**：
  > 以角色基础形象图为底图，img2img 叠加服化妆造，保持基础形象面容不变，温柔气质，
  > 【L1·妆容】正式妆；精致妆容、眼妆明显、暖粉唇色，
  > 【L2·发型】编发、发丝流畅、珠玉发饰，
  > 【L3+L4·服饰】酒红色礼服裙、丝绸材质、90 年代复古剪裁，
  > 【L5·鞋履】低跟皮鞋、与礼服统一，
  > 【L6·配饰】珍珠耳饰、细项链，
  > 四视图一致性，面容细腻渲染，自然站立姿态，暖调米白背景，图中不要有任何文字

### 3.3 L1 妆容的「先分析后决策」

这个文件有个容易被忽视的高级设计：**L1 妆容强度由系统分析剧情线索决定**，不是固定的（R12「先分析再决策」、X14「仅因道具/场景/动作词而误把妆容升级」是严禁的）。它给了「线索→妆容」映射表：仅服饰变化→基础妆；含笑/气色微提→轻妆；面色苍白→病弱妆；盛装典礼→正式妆。

**给美术指导的提醒**：如果你发现某个角色的衍生图「妆总是太浓」，别去改底模，去查这张映射表——很可能是剧情线索被误判成了「正式仪式」。你可以在表里把映射收紧。

#### 常见错误

| 错误 | 后果 |
|---|---|
| 用 text2img 重画而非 img2img 叠加 | 100% 漂脸 |
| 衍生 prompt 里写了场景/天气/手持物 | 违反 R9/R10，脸和身材变形 |
| 让角色换姿势（跑、坐、举手） | 违反 R11，姿态变了脸跟着变 |
| 底模本身就带了复杂花纹 | 衍生叠加时新旧花纹打架，画面脏 |
| 所有角色套同一套服化 | 违反 R4/原则 5「角色差异保留」，全片人物撞衫撞脸 |

## 四、`art_scene.md` 与 `art_scene_derivative.md` —— 场景

> **控制什么**：场景图的空间纵深、空气透视、季节配色、无人物约束（`art_scene.md`）；以及同一场景的景别/时段/天候/角度变体（`art_scene_derivative.md`）。
> **创作意义**：场景是「情绪的容器」。`2D` 这套第一条原则就写「场景承载情绪与叙事功能，不是纯背景板」。

### 4.1 `art_scene.md` 的关键旋钮

**景深/层次**：R1「场景必须有前中后景层次」、X3「严禁场景无纵深」。这是防「贴图感平面背景」的硬约束。模板里有 `前景：{元素}，中景：{元素}，后景：{元素}` 三段必填。

**空气透视**：R2「室外必须包含空气透视」、提示词里「空气透视，远景模糊」。这是 2D 出深度感的关键词，删了室外会变「纸片背景」。

**季节配色**：第二节给了春/夏/秋/冬四套主辅色（春=粉+嫩绿、秋=金黄+橙红……）。想换季节氛围，调这张表的主辅色即可。

**无人物约束**：R5/X5「严禁出现任何人物、人影、人体轮廓」。场景图必须是空镜，人物是另外贴上去的。这条**绝对不能删**——场景图带了人，做分镜时人物没法替换、一致性全毁。

**瑕疵感**：R8/X8「材质必须带有使用痕迹，禁止全新无瑕的 CG 感」。这是 2D 手绘风的灵魂，墙面磨损、地面痕迹让画面「活」。真人那套对应 P6「场景必须有被使用过的证据，杜绝样板间」，逻辑完全一致。

### 4.2 `art_scene_derivative.md` —— 一套场景，万种变化

衍生文件的设计哲学是「**空间结构不变，只换观察方式**」。它给了四个变化维度，每个都附「保持不变 / 允许变化」对照：

| 维度 | 取值 | 保持不变 |
|---|---|---|
| 景别 | 大全景→全景→中景→近景→特写 | 建筑外观、整体布局 |
| 时段 | 清晨/正午/黄昏/夜间月光/夜间灯火 | 建筑/布局/材质 |
| 天候 | 晴/阴/薄雾/细雨/飞雪 | 建筑/布局 |
| 角度 | 正/侧/背/俯/仰/推进/自由 | 中心点不变，仅角度切换 |

**时段变体的 Before/After**：同一个「车站站台」基准图，要黄昏版：

> 90 年代复古日系动画风格衍生场景图，基于参考图，保持场景空间结构一致，
> **黄昏 golden hour，天空色调暖化，影子拉长**，前景：站台长椅，中景：检票口，后景：远山轮廓，
> 琥珀暖色调，与参考图保持建筑结构/材质/色调一致，仅按目标时段切换，画面中无任何人物，图中不要有任何文字

注意它只动了「天空色调 + 影子」，建筑材质布局全锁死——这就是 R5「与参考图保持建筑结构/材质/色调/光线一致」。

#### 常见错误

- 衍生时连建筑结构一起改了 → 违反 X1「变体间建筑结构不一致」，同一地点像两个地方。
- 天候和季节矛盾（夏天场景配飞雪）→ X2。
- 把场景拼成多视图/网格 → R3/X6「必须单画面」。出图模型很爱自作主张拼九宫格，这条要盯紧。
- 给场景图加了人 → X5，前面说过，致命。

## 五、`art_prop.md` 与 `art_prop_derivative.md` —— 道具

> **控制什么**：道具的材质渲染、四宫格多角度设定图、纯道具静物展示（`art_prop.md`）；以及道具的状态变体——崭新/陈旧/损伤/染血/发光等（`art_prop_derivative.md`）。
> **创作意义**：信物、武器、关键道具往往是剧情的「眼」。一把会发光的剑、一封烧了一半的信，状态本身就是叙事。

### 5.1 `art_prop.md` 的关键约束

**纯道具独立展示**：第一节原则 5 + X2/X3「严禁道具与人物同画面、严禁道具被持有/佩戴/握持」。模板末尾甚至重复强调「画面中不能出现任何人物、手部、手指、肢体」。原因同场景——道具要能被独立调用、贴到任何角色手里，所以底图必须是「静物陈列」。

**四宫格规范**：第三节定义 2×2 四视图（左上正面/右上侧面/左下背面/右下细节特写）。和角色四视图、道具四宫格一样，都是为了**多角度一致性**。

**材质渲染表**：第四节给了金属/玉石/木质/陶瓷/布纸/宝石六类的渲染要求和提示词。想让某类道具质感更突出，调这张表的「渲染要求」一栏。

### 5.2 `art_prop_derivative.md` —— 状态即叙事

状态分三类：使用状态（崭新/日常/陈旧）、损伤状态（微损/破损/残片）、特殊状态（染血/浸水/燃烧/发光/包裹）。还有一张「材质状态变化规则」表，告诉你金属怎么从「亮光泽→微包浆→锈蚀」。

**Before/After：一封信物的状态推进**

- 基准（崭新）：`{信纸}，崭新、完好无损、纸质清晰`
- 衍生（燃烧）：`基于{信物}设定图，当前状态：燃烧/焦损，边缘焦黑、火烧痕迹，纸张发黄变脆，四宫格...，无人持有`

R2「状态变化须符合物理逻辑」、X2「违反物理逻辑的损伤」是它的质检线——别让金属「发霉」、让纸「生锈」。

#### 常见错误

- 道具图里出现了手 → X4/X5，违反纯道具原则，没法复用。
- 状态变了道具认不出了 → R1/X1「核心造型在所有状态中可识别」。烧到看不出是信，就失败了。
- 损伤过度血腥恐怖 → X3。

## 六、`art_storyboard_video.md` —— 分镜视频风格注入

> **控制什么**：生成**视频**提示词时，往里注入的风格标签。
> **创作意义**：图是静态的，视频要把风格「贯到动起来」。这个文件保证分镜动画段不会出戏。

这是最短的一个文件，但很关键。`2D_90s_japanese_anime/art_storyboard_video.md` 全文就一张表，按视频生成模式给不同标签：

| 模式 | 风格标签 |
|---|---|
| 通用多参模式（英文） | `90s Japanese anime, hand-drawn cel animation, soft warm tones, cinematic, clean line art, nostalgic aesthetic` |
| 通用首尾帧模式（英文） | 上面 + `shallow depth of field` |
| Seedance 2.0（中文） | `90年代日式动画，手绘赛璐璐，柔和暖调，电影风格，清晰线条，怀旧质感` |

**怎么调**：如果你换了视频模型、或发现某个模型对中文标签更敏感，就在这里加一行对应模式的标签。**注意中英文标签要语义对齐**——别中文写「怀旧」英文写「modern」，否则不同模式出来的片子调性不统一。

**常见错误**：在这里塞进一长串和图风格冲突的词（比如图是平涂、这里写 `volumetric lighting, photorealistic`）→ 视频段和静帧段对不上，观众一眼看出「换皮了」。**视频标签应是静帧风格的「动态版同义改写」，不是新风格**。

## 七、`driector_skills/` 三个文件 —— 导演级视觉规划

这三个文件不进出图 prompt，而是被**制作 Agent 通过 `activate_skill` 加载**，用来做「导演决策」——全局怎么打光、分镜面板怎么构图、分镜表怎么排节奏运镜。它们有 frontmatter（`name` + `description`），这点和 `art_prompt/` 下的文件不同，**改它们时要特别小心 `name` 字段**（见第八节）。

### 7.1 `director_planning_style.md` —— 全局视觉规划

> **控制什么**：全片层面的色调体系、光影方案（A–F 六套）、质感方向、场景空间元素的叙事用法、配乐与环境音。

最值得调的是**光影方案表**——它把情绪段落映射到一套打光方案，是 `prefix.md` 情绪色盘的「执行版」：

| 情绪段落 | 光影方向 | 色调倾向 | 适用阶段 |
|---|---|---|---|
| 心动瞬间 | B·暖调侧光 | 樱花粉 + 暖橙局部 | 升温段 |
| 黄昏浪漫 | D·逆光暖调 | 琥珀暖主导 + 冷蓝背景 | 甜蜜段 |
| 夜晚月色 | E·月光冷调 | 淡蓝主调 + 暖色局部点缀 | 夜晚段 |

**联动提醒**：第一节我们改 `prefix.md` 情绪色盘时说过，这里要同步。比如你在情绪色盘新增了「暗恋嫉妒」，就该在这张光影方案表里也加一行对应的打光方向（如「G·冷压侧光」），否则导演 Agent 规划到这个情绪时会找不到打光依据，只能瞎猜。

它还有很「美术指导」的两段：**场景空间元素的叙事功能**（窗框/门框=框架构图、樱花=心动、落叶=离愁），以及**留白原则**（「关键情感瞬间优先去掉配乐只留环境音」「全片配乐覆盖率不超过 50%」）。这些是给 Agent 的「审美口味」，想让片子更克制/更满，调这里。

真人那套 `realpeople_modern_city/driector_skills/` 对应的会是「摄影机语法的导演化」——焦段选择、机位运动的叙事心理，思路一致但词汇是电影摄影的。

### 7.2 `director_storyboard.md` —— 分镜面板视觉技法

> **控制什么**：单个分镜画面的视觉技法——情绪→面容/眼神词映射、光影氛围词库、场景质感约束词、固定风格锚定词、负向词模板、完整生成示例。

这是三个里最「实操」的，几乎是一本「分镜提词词典」。几个关键旋钮：

- **情绪→面容/眼神词映射表**：把「心动」翻译成「嘴角微扬，脸颊微红 / 眼神明亮，目光温柔 / 眼睛弯月形」。想调整全片角色的「表演分寸」，改这张表。
- **场景质感约束词**：按场景类型给必加词（日式学校→「木质地板纹理，黑板粉笔字，窗外绿树」）。想让某类场景细节更足，往这里加词。
- **固定风格锚定词 + 画质锁定词**：所有输出必带的句子。这是分镜段的「风格保险丝」。
- **负向词模板**：注意它特别标注「⚠️ Seedream（模式 A）不支持负向提示词，负向词仅适用于模式 B」。**这是踩过坑才会写的提醒**——给不支持负向词的模型塞负向词是无效甚至有害的。你换模型时一定要确认这点。

它还附了**完整生成示例**（模式 A Seedream 中文 / 模式 B Nanobanana 的 XML 结构），直接展示了「分镜表行数据 → 最终 prompt」的转换。改这个文件时，照着示例的结构改最稳。

### 7.3 `director_storyboard_table_style.md` —— 分镜表节奏与运镜

> **控制什么**：分镜表层面的节奏约束——同场戏光影统一、光源角度、动作节奏、环境动态、运镜禁忌、转场禁忌。

精华是**运镜与转场禁忌**：

- 禁用快速运镜（甩镜、急推、手持晃动）——和 90s 日式动画气质冲突
- 禁用花式转场（划屏、旋转、百叶窗）
- 环境动态：每 3–4 个镜头至少一个有动态的镜头（樱花飘落、电车驶过），避免画面「死」掉

**这是控制「片子节奏」的旋钮**。如果你的项目是快节奏都市悬疑，90s 这套的「禁快速运镜」就不合适——但这恰恰说明你该用真人都市风格，那套的运镜禁忌是另一套逻辑（手持微晃是被鼓励的「在场感」）。**不要在 90s 这套里强行解禁快速运镜**，会和它整体气质打架。

## 八、从零新建一个自定义画风

当内置 11 套都不对味，或者你要做一套强烈个人风格时，**复制改造**远比从头写省事。流程如下：

### 8.1 步骤

1. **复制一套最接近的目录**作为模板。比如要做「2D 水彩绘本风」，从 `2D_flat_design` 或 `2D_90s_japanese_anime` 复制：
   `cp -r data/skills/art_skills/2D_90s_japanese_anime data/skills/art_skills/2D_watercolor_storybook`
2. **改 `prefix.md`**：换风格基因表、色板 hex、情绪色盘、色温饱和度、R/X 约束。这是工作量最大的一步。
3. **改 7 个 `art_prompt/` 文件**：把里面的「90s anime style / 手绘平涂」等锚词全局替换成你的风格锚词；保留所有「四视图/四宫格/无人物/img2img/不裁切」这类**结构性硬约束**——它们和风格无关，是系统正常工作的前提。
4. **改 3 个 `driector_skills/` 文件**：改技法描述。**最关键的是改 frontmatter 的 `name`**（见 8.2）。
5. **改 `README.md`**：给人看的说明，同步更新。
6. **改 `art_storyboard_video.md`**：换视频风格标签。

### 8.2 必须齐全的文件清单

一套合格的风格目录，下面这些**一个都不能少**（系统按文件名查找，缺了对应资产类型就出不了图或回退到只有 prefix）：

```
2D_watercolor_storybook/
├── README.md                          （建议有，给人看）
├── prefix.md                          （必须，全局拼接）
├── art_prompt/
│   ├── art_character.md               （必须）
│   ├── art_character_derivative.md    （必须）
│   ├── art_prop.md                    （必须）
│   ├── art_prop_derivative.md         （必须）
│   ├── art_scene.md                   （必须）
│   ├── art_scene_derivative.md        （必须）
│   └── art_storyboard_video.md        （必须）
└── driector_skills/
    ├── director_planning_style.md     （必须，含 frontmatter）
    ├── director_storyboard.md         （必须，含 frontmatter）
    └── director_storyboard_table_style.md（必须，含 frontmatter）
```

> 备注：`getArtPrompt` 是递归按文件名查找的（`src/utils/getArtPrompt.ts` 的 `findFileRecursive`），所以子目录层级可以容错，但**文件名必须精确匹配**（`art_character.md` 不能写成 `character.md`）。

### 8.3 frontmatter 怎么写（最容易翻车的地方）

`driector_skills/` 下三个文件**必须**带 frontmatter，格式：

```markdown
---
name: watercolor_storybook_planning_style
description: 水彩绘本约束 — 定义本风格在色调、光影、质感上的全局约束。适用于任何叙事类型。
metaData: director_skills
---
```

**铁律**（来自 `src/utils/agent/skillsTools.ts`）：

1. **必须有 `name` 和 `description` 两个字段**，缺一个直接抛错（`parseFrontmatter` 第 114 行）：「技能文件缺少必要字段: name 或 description」。
2. **`name` 在所有被加载的技能里必须唯一**。Agent 通过 `activate_skill` 用 `z.enum(skillNames)` 精确匹配（第 191 行），重名会导致匹配错乱。所以**复制目录后，一定要把三个文件的 `name` 改成你新风格专属的名字**——别让 `2D_watercolor_storybook` 里还留着 `director_planning_style`（虽然内置风格之间靠目录隔离，但同一次加载里出现重名 `name` 会冲突）。
3. 观察内置规律：`2D_90s_japanese_anime` 用的是 `name: director_planning_style`（通用名），而 `realpeople_modern_city` 用的是 `name: liveaction_urban_character`、`liveaction_urban_storyboard`（带风格前缀）。**推荐学真人那套，给 `name` 加风格前缀**，天然避免重名，例如 `watercolor_planning_style`、`watercolor_storyboard`、`watercolor_storyboard_table_style`。
4. `description` 会被拼进 Agent 的技能列表展示（`buildSkillPrompt`），Agent 靠它判断「要不要激活这个技能」。所以 `description` 要写清「这是什么风格的什么技法、适用什么场景」，写得含糊 Agent 可能不激活它。

### 8.4 新建后的自检清单

- [ ] 11 个文件齐全、文件名精确匹配
- [ ] `prefix.md` 的色板、色温、R/X 已全部本风格化
- [ ] 7 个 `art_prompt` 文件的风格锚词已替换，结构性硬约束（四视图/无人物/img2img/不裁切）已保留
- [ ] 3 个 `driector_skills` 文件 frontmatter 的 `name` 已改成本风格专属且唯一
- [ ] `art_storyboard_video.md` 视频标签已换且中英文语义对齐
- [ ] 跑一个角色四视图 + 一个场景图，确认风格、不裁切、无漂脸

## 九、「想要某种视觉效果 → 改哪个文件哪一段」对照表

| 我想要的效果 | 改哪个文件 | 改哪一段 |
|---|---|---|

| 整片饱和度太浓/太淡 | `prefix.md` | 「色温约束」表的饱和度区间 |

| 角色总是漂脸 | `art_character_derivative.md` | 确认 img2img + 面容不变(R1) + 姿态锁定(R11) + 范畴隔离(R9/R10) |

## 十、风格 × 题材搭配的视觉建议

选错风格，再精调也是逆水行舟。下面是按内置 11 套给的搭配建议（媒介从风格目录名前缀判断：`2D_` 手绘、`3D_` 三维渲染、`realpeople_` 真人实拍）：

| 题材 | 首选风格 | 理由 / 提醒 |
|---|---|---|
| 校园/纯爱/治愈 | `2D_90s_japanese_anime` | 暖怀旧基因 + 留白配乐哲学天生适配；切忌强行加快节奏运镜 |
| 古言/仙侠/国风言情 | `2D_chinese_guofeng` / `3D_chinese_traditional` | 要不要立体感决定 2D 还是 3D |
| 都市职场/成长/轻悬疑 | `realpeople_modern_city` / `realpeople_urban_modern` | 真人实拍最有代入感；记牢「皮肤像硅胶=越界」红线 |
| 成熟向都市情感 | `2D_mature_urban_romance` | 2D 但偏写实成熟，避开 90s 的少年感 |
| 科幻/赛博/未来国风 | `3D_guofeng_cyber` | 唯一允许霓虹/高饱和的风格——别在别的风格里硬塞 |
| 知识科普/品牌/极简叙事 | `2D_flat_design` | 扁平设计信息清晰；不适合强情绪剧情 |
| 童话/手作温暖向 | `3D_clay_stopmotion` | 黏土定格质感独特；产能和一致性要预留余量 |
| 古装真人剧情 | `realpeople_ancient_chinese` | 真人 + 古装；不要试图改 `realpeople_modern_city` 的 X3 去硬凑 |

**跨风格混用的提醒**：一个项目原则上锁定一套风格到底。确实需要「现代都市闪回到过去」这种跨时代叙事时，正确做法是**派生一套子风格**（复制目录、在 prefix 里放宽题材边界并明确标注闪回段落），而不是删主风格的严禁项——保持内置风格的纯净，是为了让团队其他项目还能复用。

## 本章调优速查表

| 旋钮 | 文件 / 段落 | 改它的效果 | 头号坑 |
|---|---|---|---|
| 色板 hex | `prefix.md` · 色板 C1–C10 | 改片子色彩人格 | 引入荧光色撞严禁项；只改色名不改 hex |
| 情感配色 | `prefix.md` · 情绪色盘 | 色彩叙事对齐情绪 | 单镜头超 2 主色；用色板外的色 |
| 色温/饱和 | `prefix.md` · 色温约束 | 改全片冷暖浓淡 | 拉冷拉透会撞「现代化」严禁项，应换风格 |
| 全局 R/X | `prefix.md` · 必守/严禁 | 风格免疫系统 | 删媒介锚 → 风格崩；删 X → 多废图 |
| 角色气质 | `art_character.md` · 肤感/体型 | 调底模默认气质 | 删四视图「不裁切」「一致性」约束 |
| 防漂脸 | `art_character_derivative.md` | 换装不换脸 | 用 text2img；带场景/道具/换姿势 |
| 妆容强度 | `art_character_derivative.md` · 线索映射 | 控制上妆分寸 | 误把场景/道具线索当正式妆升级 |
| 场景纵深 | `art_scene.md` · 前中后景/空气透视 | 防贴图感平面 | 场景图里出现人物（致命） |
| 场景变体 | `art_scene_derivative.md` | 一景多时段/天候 | 改了建筑结构；被拼成九宫格 |
| 道具材质/状态 | `art_prop*.md` | 质感与叙事状态 | 道具被手持/带人；状态变到认不出 |
| 视频风格 | `art_storyboard_video.md` | 动态段风格统一 | 中英标签语义不一致；塞冲突词 |
| 全局光影 | `director_planning_style.md` · 光影方案 | 导演级打光基调 | 改情绪色盘没同步改光影方案 |
| 表演/分镜词 | `director_storyboard.md` | 表情、场景细节、画质 | 给不支持负向词的模型塞负向词 |
| 节奏运镜 | `director_storyboard_table_style.md` | 片子节奏 | 在 90s 风格里强行解禁快速运镜 |
| 新建风格 | 复制整个目录 | 全新画风 | `driector_skills` frontmatter `name` 漏改/重名 → Agent 报错 |

> **最后一句**：视觉手册的设计哲学是「**风格基因可调，结构约束不可删**」。色值、情绪、光影、气质这些「味道」尽管大胆调；但四视图不裁切、img2img 防漂脸、场景无人物、道具不被持有、frontmatter 的 `name` 唯一——这些是系统能正常工作的骨架，删一根，整套就垮。调味自由，拆骨头要命。
