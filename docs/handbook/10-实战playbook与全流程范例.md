# 第10章 实战 Playbook：四个题材的端到端范例

## 本章导览

前面几章把每个字段、每个环节都拆开讲透了。但真正动手时，你需要的是"照着做"的整套配方——从立项到成片，一个题材一条龙。

这一章给你四个端到端实战范例，覆盖最常见的短剧/漫剧题材：

1. **都市重生爽剧**——快节奏、强打脸、真人质感；
2. **古风仙侠**——国风画面、宿命叙事、多主体同框；
3. **悬疑烧脑**——冷峻氛围、信息控制、节奏克制；
4. **甜宠言情**——情绪拉扯、柔和画面、对话密集。

每个范例都按同一套骨架走：**项目字段配置 → 剧本阶段要点 → 视觉/导演手册重点 → 角色资产 describe 范例 → 分镜与视频 mode 选择 → 配音 → 成片注意**。

章末附两份硬通货：**全流程检查清单**（开工到收尾逐项打勾）和**全局调优速查总表**（想改 X → 动哪个字段/文件/操作）。

> 阅读前提：本章引用的字段（`type`/`intro`/`artStyle`/`directorManual`/`videoRatio`/`videoModel`/`mode`）在第01章已逐字段拆解；视频 `mode` 和提示词机制在第08章。这里不重复原理，只给"该怎么配"的实战值。所有 `artStyle`/`directorManual` 取值对应 `data/skills/art_skills/` 和 `data/skills/story_skills/` 下的真实文件夹名。

## 范例一：都市重生爽剧

> 关键词：被退婚/被羞辱 → 重生/隐藏身份 → 逐步打脸碾压。爽点密度是命门。

### 1. 项目字段配置

| 字段 | 推荐值 | 理由 |
|---|---|---|
| `name` | `《重生之我要当首富》` | 用真书名，给 AI 题材锚点 |
| `type` | `都市重生爽文 / 隐藏身份逆袭，强打脸快节奏；每 2-3 集一次身份揭穿高潮` | 不只写"都市"，把套路+节奏说清 |
| `artStyle` | `realpeople_modern_city` | 真人都市最增强"打脸"的代入与解气 |
| `directorManual` | `Urban_workplace_drama` | 职场/都市叙事节奏，镜头干脆 |
| `videoRatio` | `9:16` | 短剧主战场是竖屏 |
| `imageQuality` | `2K`（封面/主视觉 `4K`） | 常规分镜 2K 足够 |
| `videoModel` | 选支持图生视频 + 多参的模型 | 见 mode 选择 |
| `mode` | `["imageReference:N"]`（N≥2）或 `singleImage` | 对戏镜头多参，单人镜头单图 |

### 2. 剧本阶段要点

- **`intro` 写成立项书**（见第01章）：主线一句话（谁、因为什么、要干成什么、最大障碍）+ 爽点节奏（每 2-3 集一个打脸高潮）+ 人物钩子（主角的"反差"：表面落魄、实则碾压）+ 基调（爽快、解气、忌拖沓）。
- **节奏**：开篇 30 秒内必须出冲突（被羞辱/被退婚），别铺垫。
- **钩子**：每集结尾留反转。

### 3. 视觉 / 导演手册重点调哪些段

- 视觉手册 `realpeople_modern_city`：重点确认"现代都市质感、商务/写字楼场景、人物着装现代"的描述与你的设定一致。
- 导演手册 `Urban_workplace_drama`：关注"对峙镜头、打脸时刻的景别与停留"——爽剧的打脸要给特写、要给反应镜头。

### 4. 角色资产 describe 范例

资产 `describe` 是 30-80 字的视觉化描述（见资产提取规范）。男主示例：

> 三十岁左右男性，身形挺拔，深色高定西装，发型利落，眼神冷峻锐利，下颌线分明，气场沉稳带压迫感；表面低调克制，实则强势。

反派（被打脸对象）示例：

> 四十岁男性，微胖，名牌却俗气的金表与西装，神态傲慢，常带轻蔑笑意，眼神浮夸。

### 5. 分镜与视频 mode 选择

| 镜头类型 | 推荐 mode | 提示词重点 |
|---|---|---|
| 男主单人特写（说狠话） | `singleImage` | 运镜：缓慢推近；表情：冷峻；台词标 dialogue |
| 男主 vs 反派对峙同框 | `imageReference:N`（N≥2，两人图都传） | 双人构图、对峙张力、谁压谁的站位 |
| 打脸高潮反应镜头 | `singleImage` | 反派震惊表情、短时长（3s）快切 |
| 场景空镜（写字楼/豪车） | `singleImage` 或 `text` | 氛围、光影 |

### 6. 配音

- 男主绑低沉、有压迫感的音色；反派绑略浮夸的音色——音色反差强化打脸爽感。
- 用 `batchBindAudio` 自动匹配后人工复核（`updateAssetsAudio` 改绑）。
- 对峙台词镜头若要对口型：模型需支持 + `mode` 含 `audioReference:K` + 生成时 `audio: true`。
- 内心独白（"哼，一群井底之蛙"）标 OS，按旁白处理，不需对口型。

### 7. 成片注意

- 打脸节奏靠**剪辑节奏**：狠话特写 → 反派震惊反应 → 围观者哗然，快切。
- 关键打脸镜头多抽 2-3 个候选挑表情最到位的。
- BGM（爽感配乐）、字幕、转场留外部剪辑。

## 范例二：古风仙侠

> 关键词：血仇/问道 → 境界突破 → 宿命对决。画面要"仙气"，叙事要"宿命感"。

### 1. 项目字段配置

| 字段 | 推荐值 | 理由 |
|---|---|---|
| `name` | `《问道凌霄》` | — |
| `type` | `古风仙侠 / 复仇问道，热血带宿命悲剧感；境界突破需有代价，忌无脑升级` | 点明基调与边界 |
| `artStyle` | `2D_chinese_guofeng` 或 `3D_guofeng_cyber` | 国风画面是仙侠的根 |
| `directorManual` | `Xianxia_fantasy` | 仙侠叙事节奏与镜头语言 |
| `videoRatio` | `9:16` | — |
| `imageQuality` | `2K`（飞升/法宝等主视觉 `4K`） | — |
| `videoModel` | 支持多参的模型（多主体/法宝同框多） | — |
| `mode` | `["imageReference:N"]`（N 给够，仙侠同框元素多） | 角色+场景+法宝常同框 |

### 2. 剧本阶段要点

- `intro` 强调"目的不是变强而是揭真相"这类内在动机，避免纯升级流。
- 每个境界突破要有代价/牺牲——这是仙侠脱离俗套的关键，写进 intro 的"边界"。
- 双线情感（亦敌亦友）可作钩子。

### 3. 视觉 / 导演手册重点

- `2D_chinese_guofeng`：确认服饰（道袍/法衣）、场景（山门/秘境/云海）、色调（青绿/水墨）描述到位。
- `Xianxia_fantasy`：关注"法术/飞行/对决"的运镜倾向——仙侠需要大开大合的运镜和特效感。

### 4. 角色资产 describe 范例

主角示例：

> 少年修士，约二十岁，身形清瘦挺拔，墨发束冠，眉目清俊带一丝冷意，身着青色道袍，腰悬古朴长剑，气质孤傲清冷，眼底藏复仇执念。

法宝（道具 tool）示例：

> 古朴长剑，剑身泛幽蓝寒光，剑格雕云龙纹，剑穗暗红，出鞘时附淡淡剑气流光。

### 5. 分镜与视频 mode 选择

| 镜头类型 | 推荐 mode | 提示词重点 |
|---|---|---|
| 角色御剑飞行/法术 | `imageReference:N`（角色+法宝图） | 运镜：环绕/拉远；光效；衣袂飘动 |
| 境界突破/变身 | `startEndRequired`（突破前/后两帧） | 首帧凡人态、尾帧仙态，AI 补过渡 |
| 双人对决同框 | `imageReference:N`（两角色图） | 对峙气场、剑气交锋 |
| 云海/山门空镜 | `singleImage` 或 `text` | 仙气氛围、光影层次 |

> 仙侠的"变身/突破"镜头是 `startEndRequired` 的绝佳用例——给出突破前后两个画面，让模型生成过渡，比单图硬生成更可控。

### 6. 配音

- 主角绑清冷少年音；师长/反派绑沉稳或苍老音色。
- 仙侠台词多文言/半文言，注意 TTS 对生僻字的发音——配音前试听，不对就换音色或微调文本。
- 法术吟唱、剑鸣等音效写在分镜 `videoDesc` 音效字段，由模型尽量体现（精细音效仍需外部补）。

### 7. 成片注意

- 突破/对决高潮镜头优先多抽候选，特效随机性大。
- 仙侠节奏可比爽剧稍慢，给情绪和氛围留时间，但单镜头别超 8s 防发散。
- 特效字幕（境界名/招式名）、BGM 外部处理。

## 范例三：悬疑烧脑

> 关键词：连环案/调查 → 线索揭示+反转 → 真凶后置。氛围冷峻，信息要"挤牙膏"。

### 1. 项目字段配置

| 字段 | 推荐值 | 理由 |
|---|---|---|
| `name` | `《雾隐十年》` | — |
| `type` | `都市悬疑 / 连环案，智性冷峻；线索揭示+反转驱动，真凶藏到后三分之一，忌巧合推进` | — |
| `artStyle` | `realpeople_modern_city` | 真人冷峻质感最适合悬疑 |
| `directorManual` | `Mystery_thriller` | 悬念叙事节奏、信息控制 |
| `videoRatio` | `9:16` | — |
| `imageQuality` | `2K` | — |
| `videoModel` | 图生视频模型即可 | 悬疑多单人/对话镜头 |
| `mode` | 以 `singleImage` 为主，对话同框用 `imageReference:N` | — |

### 2. 剧本阶段要点

- `intro` 的钩子机制要写清："每集抛一个新疑点、收一个旧疑点"，真凶后置。
- 强调"忌过早泄底、忌巧合推动"——这是悬疑的两大死法，写进边界。
- 人物钩子：调查者本身可能是知情者/受害者，双重身份制造张力。

### 3. 视觉 / 导演手册重点

- `realpeople_modern_city`：确认"冷色调、阴影、夜景、雨"等氛围元素。
- `Mystery_thriller`：关注"悬念镜头的景别与停留"——悬疑爱用特写（手、眼、关键道具）和缓慢运镜制造不安。

### 4. 角色资产 describe 范例

刑警（主角）示例：

> 三十五岁女性，干练短发，眼神锐利疲惫，深色风衣，神态冷静克制，眉间常蹙，透出长期办案的压力感。

关键道具（线索物）示例：

> 一枚褪色的旧怀表，表盖刻模糊字母，链条断裂，表面有划痕，停在某个特定时刻。

### 5. 分镜与视频 mode 选择

| 镜头类型 | 推荐 mode | 提示词重点 |
|---|---|---|
| 线索道具特写 | `singleImage` | 极近景别、缓慢推近、冷光、短时长聚焦 |
| 审讯/对话对峙 | `imageReference:N`（双人） | 对视张力、明暗对比 |
| 主角独自推理（内心戏） | `singleImage` | 静态偏多、台词标 OS（内心独白） |
| 案发现场空镜 | `singleImage` | 阴郁氛围、留白、不安感 |

### 6. 配音

- 主角绑冷静低沉音色；旁白/内心独白大量用 OS 标注（悬疑内心戏多）。
- 注意：OS/VO **不需要对口型**，按旁白配即可——这能省掉对口型的麻烦，悬疑反而更适合"画面+旁白"的克制风格。
- 关键反转处的台词，配音节奏要"压住"，靠停顿制造悬念。

### 7. 成片注意

- 悬疑靠**节奏克制**：信息别一次给完，镜头停留可稍长营造压抑，但单镜头别超 8s。
- 线索特写镜头多抽候选，要"看清又看不全"的感觉。
- 字幕（尤其旁白/内心独白）很重要，外部烧字幕时分清 dialogue/OS/VO。
- BGM 用低频氛围音，外部处理。

## 范例四：甜宠言情

> 关键词：相遇/误会 → 拉扯/心动 → 双向奔赴。情绪密度是命门，画面要"甜"。

### 1. 项目字段配置

| 字段 | 推荐值 | 理由 |
|---|---|---|
| `name` | `《偏偏喜欢你》` | — |
| `type` | `都市甜宠言情 / 双向奔赴，情感拉扯节奏；甜中带虐、忌狗血，单集一个心动名场面` | — |
| `artStyle` | `2D_mature_urban_romance` | 柔和都市言情画风最"甜" |
| `directorManual` | `Sweet_romance_novel` | 情感拉扯节奏、心动镜头语言 |
| `videoRatio` | `9:16` | — |
| `imageQuality` | `2K` | — |
| `videoModel` | 图生视频 + 多参（双人对手戏多） | — |
| `mode` | `["imageReference:N"]`（N≥2，男女主常同框） | — |

### 2. 剧本阶段要点

- `intro` 钩子：每集一个"心动名场面"（壁咚、雨中、误会解开）。
- 基调"甜中带虐、忌狗血"——言情最容易翻车在狗血，写进边界。
- 人物钩子：男女主的"反差萌"或"双向暗恋"是甜点核心。

### 3. 视觉 / 导演手册重点

- `2D_mature_urban_romance`：确认"柔光、暖色调、唯美构图、人物造型精致"。
- `Sweet_romance_novel`：关注"心动瞬间的镜头停留与特写"——言情靠眼神、靠近距离、靠氛围光。

### 4. 角色资产 describe 范例

女主示例：

> 二十三岁女性，长发微卷，五官精致温柔，眼神清澈带笑意，米色针织衫与半裙，气质温婉又带点小俏皮。

男主示例：

> 二十七岁男性，身形修长，浅色衬衫，发型清爽，眉眼深邃带宠溺笑意，气质温润又略带霸道。

### 5. 分镜与视频 mode 选择

| 镜头类型 | 推荐 mode | 提示词重点 |
|---|---|---|
| 男女主对视/靠近 | `imageReference:N`（两人图） | 近距离构图、暖光、心动氛围、轻微运镜 |
| 单人心动反应（脸红/微笑） | `singleImage` | 特写、表情细腻、柔光 |
| 名场面（壁咚/拥抱） | `imageReference:N` | 双人姿态、氛围拉满 |
| 浪漫空镜（樱花/咖啡馆/雨） | `singleImage` 或 `text` | 唯美氛围、暖色 |

### 6. 配音

- 女主绑温柔甜音；男主绑温润低音。言情对音色"甜度"最敏感，多试听几条候选音色再绑。
- 对话密集，台词大多是 dialogue（需对口型时按第08章三条件配齐）。
- 心动旁白/内心独白用 OS。

### 7. 成片注意

- 甜宠靠**情绪铺陈**：心动镜头可适当慢、给停留，但单镜头别超 8s。
- 名场面镜头多抽候选挑氛围最对的。
- 暖色调要全片统一，画幅/画风别中途变。
- BGM（甜系/钢琴）、字幕、转场外部处理。

## 全流程检查清单（开工到收尾）

### A. 立项阶段（第01章）

- [ ] `name` 用真实书名，不是测试名/编号
- [ ] `type` 写到"大类+套路+基调"精度
- [ ] `intro` 写成立项书四件套（主线/爽点/人物钩子/基调与边界），150-400 字
- [ ] `artStyle` 选对题材的画风文件夹（**出图前定死，改一次=重做资产**）
- [ ] `directorManual` 与题材同向（别误当画风选）
- [ ] `videoRatio` 短剧/漫剧设 `9:16`（别沿用默认 16:9）
- [ ] `videoModel` 与 `mode` 能力自洽（多参模型才配多参 mode）
- [ ] `imageQuality` 常规 2K，主视觉 4K

### B. 剧本与资产阶段

- [ ] 剧本主线/爽点与 `intro` 一致（不打架）
- [ ] 资产提取齐全（角色/场景/道具），`describe` 是 30-80 字视觉化描述
- [ ] 角色资产图出图、人物造型一致
- [ ] 音频资产已上传（否则无法绑音色）
- [ ] 角色音色已绑定（`batchBindAudio` 后人工复核）

### C. 分镜与提示词阶段（第08章）

- [ ] 分镜 `videoDesc` 12 字段填全（尤其**台词**和**台词类型** dialogue/OS/VO）
- [ ] 每个分镜挂对了关联资产（`o_assets2Storyboard`）
- [ ] 视频提示词生成成功（`o_videoTrack.state=已完成`）
- [ ] 关键镜头手改提示词（`updateVideoPrompt`）加运镜/景别/时间分段
- [ ] `duration` 与台词长度匹配（一句话念几秒，配多长视频）

### D. 视频生成阶段

- [ ] `duration`/`resolution` 组合在模型 `durationResolutionMap` 内
- [ ] 多主体镜头用了多参 mode 且 N 给够
- [ ] 转场/变身镜头用了 `startEndRequired`
- [ ] 对口型镜头：模型支持 + `mode` 含 `audioReference:K` + `audio: true` 三条齐
- [ ] 批量生成分批进行（每批十几条，防撞供应商速率限制）
- [ ] 失败镜头看 `errorReason`/`reason` 后人工重生成（无自动重试）

### E. 选片与成片阶段

- [ ] 关键镜头抽了 2-3 个候选挑最佳
- [ ] 每条 track 都用 `selectVideo` 选定了 `videoId`（无遗漏"未选"）
- [ ] 分镜 `index` 顺序正确（决定成片镜头顺序）
- [ ] 画幅全片统一
- [ ] 导出后外部补：字幕（按 dialogue/OS/VO）、BGM、音效、转场、精剪

## 全局调优速查总表（想改 X → 动哪个字段/文件/操作）

| 想改的东西 | 动哪里 | 字段/路由/文件 | 生效阶段 |
|---|---|---|---|
| 剧情主线/爽点跑偏 | `intro` | `o_project.intro` / `addProject.ts` | 剧本 Agent |
| 节奏/套路不像该类型 | `type`（必要时配 `directorManual`） | `o_project.type` | 剧本 / 制作 |
| 画面风格/人物不对味 | `artStyle` | `o_project.artStyle` → `data/skills/art_skills/` | 制作（也影响剧本语境） |
| 运镜/镜头叙事感 | `directorManual` | `o_project.directorManual` → `data/skills/story_skills/` | 制作 Agent |
| 成片黑边/比例 | `videoRatio` | `o_project.videoRatio` | 制作（出图/出片） |
| 图糊/不够清晰 | `imageQuality` | `o_project.imageQuality` | 制作（出图） |
| 出图/出片整体质量 | `imageModel` / `videoModel` | `o_project.*Model` | 制作 |
| 参考方式（单图/首尾帧/多参/音频） | `mode` | `o_project.mode` | 制作（决定提示词模板+参考） |
| 资产长相/设定 | 资产 `describe`/`prompt` + 重出图 | `o_assets` / 资产中心 | 制作 |
| 角色用哪条嗓子 | 音色绑定 | `o_assetsRole2Audio` / `batchBindAudio.ts` / `updateAssetsAudio.ts` | 配音 |
| 分镜内容/台词/景别/运镜 | 分镜 `videoDesc` 12 字段 | `o_storyboard` | 分镜 |
| 单镜头视频提示词（手改） | `updateVideoPrompt` | `updateVideoPrompt.ts` | 工作台 |
| 镜头时长/节奏 | `duration` | `o_videoTrack.duration` / `updateVideoDuration.ts` | 工作台 |
| 视频提示词模板本身 | 改模板文件 或 `o_modelPrompt` 绑定 | `data/modelPrompt/video/*.md` | 制作 |
| 增删镜头轨道 | addTrack / deleteTrack | `addTrack.ts` / `deleteTrack.ts` | 工作台 |
| 一镜选哪个候选 | selectVideo | `selectVideo.ts` | 工作台 |
| 删候选视频 | delVideo | `delVideo.ts` | 工作台 |
| 批量出片并发 | `concurrentCount`（提示词）/分批（视频） | `batchGeneratePrompt.ts` / `batchGenerateVideo.ts` | 工作台 |
| BGM/字幕/转场/精剪 | 外部剪辑软件 | —（工作台不支持） | 成片后 |

> 黄金法则（贯穿全书）：**"讲什么"的问题往剧本字段找（name/type/intro）；"长什么样、怎么拍"的问题往制作字段找（artStyle/directorManual/画幅/模型/mode/duration）；"声音"的问题往音色绑定找；"成片包装"（字幕/BGM/转场）目前在工作台外。** 定位对了层，改一次就到位，不用在对话框里反复返工。
