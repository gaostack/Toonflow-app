# 第 07 章 · 角色资产与分镜出图

> 写给创作者的实战指南。本章只讲一件事：**怎么让 AI 稳定地画出"同一个人、同一种画风、同一套世界观"的图**，并把它正确地塞进每一个镜头里。

## 本章导览

漫剧的画面质量，90% 取决于你在"资产"这一步把地基打得多牢。Toonflow 的出图链路设计得很克制：你只填一个 `describe`（描述），系统会把它喂给一段美术约束模板、生成出图提示词 `prompt`，再交给图像模型出图。听起来简单，但真正决定成败的，是你**描述里写了什么、没写什么**。

本章我会按真实代码链路，把这套机制拆给你看：

- 资产到底是什么、字段含义，以及 role/scene/tool 的写法差异；
- `describe` 字段的写法工程——基础角色写全、衍生角色只写变化量（**这是零成本、最高收益的环节**）；
- 角色一致性的底层原理：父级描述 + 父级图 img2img 是怎么"锁脸"的，为什么衍生里重复写长相反而会"漂脸"；
- "Prompt 重新生成"的完整依赖链，以及想让画面更合理时**该按什么顺序去调**；
- `imageQuality`（1K/2K/4K）与 `aspectRatio` 的取舍；
- 分镜出图：关联资产如何作参考图、怎么保镜头一致、失败怎么重试；
- 一张"症状→病因→改哪里"的排障表。

涉及的核心代码：

| 环节 | 文件 |
|---|---|
| 资产/角色出图入口 | `src/routes/production/assets/batchGenerateAssetsImage.ts` |
| 分镜出图入口 | `src/routes/production/storyboard/batchGenerateImage.ts` |
| 美术模板加载器 | `src/utils/getArtPrompt.ts` |
| 画风模板（示例画风） | `data/skills/art_skills/2D_90s_japanese_anime/art_prompt/*.md` |
| 全局美学前缀 | `data/skills/art_skills/<画风>/prefix.md` |
| 资产/图片表结构 | `src/lib/initDB.ts`（`o_assets` / `o_image` / `o_storyboard` / `o_assets2Storyboard`） |

## 一、先搞清楚"资产"是什么

所有角色、场景、道具，在数据库里都是 `o_assets` 表的一行。和出图最相关的字段（见 `src/lib/initDB.ts`）：

| 字段 | 含义 | 谁来填 |
|---|---|---|
| `name` | 资产名（如"林晚秋"） | 你 / Agent |
| `describe` | **资产描述**——你用自然语言写的设定 | **你**（最关键） |
| `type` | 类型：`role` 角色 / `scene` 场景 / `tool` 道具 / `clip` | Agent |
| `prompt` | **AI 生成的出图提示词**（由 describe 经模板加工而来） | 系统自动写回 |
| `assetsId` | 父资产 ID。衍生资产指向其基础资产；基础资产为 `null` | 系统 |
| `imageId` | 关联 `o_image` 表的图片记录 | 系统 |

出图后落到 `o_image` 表：`filePath`（图路径）、`state`（生成中 / 已完成 / 生成失败）、`model`、`resolution`、`errorReason`。

**关键认知：**

1. **你直接控制的只有 `describe`。** `prompt` 是系统拿你的 `describe` 跑一遍 AI 生成的，不是你手填的。想改 prompt，本质是改 describe（或改模板）。
2. **资产分两层**：`assetsId` 为空的是**基础资产**（角色底模/场景主图/道具主图）；`assetsId` 指向父级的是**衍生资产**（同一角色换装、换妆、换造型）。这个父子关系，是整套一致性机制的命根子。

## 二、describe 字段的写法工程（最重要、零成本）

这一节请反复读。你不需要懂 prompt engineering，只需要学会**像写人物设定卡一样写 describe**。系统会替你翻译成专业出图提示词。

### 2.1 基础角色：把"这个人长什么样"写全

基础角色（`assetsId` 为空、`type=role`）是整部剧所有镜头一致性的**源头**。它出的那张四视图设定图，会被后面所有衍生资产和分镜当参考图反复引用。**这张图错了，全剧跟着错。**

所以基础角色的 describe 要写全五个维度：

| 维度 | 要写什么 | 反例（漏写的后果） |
|---|---|---|
| **五官 / 脸** | 脸型、眼型、眉、鼻、唇、有无特征（痣/疤/雀斑） | 不写→每次脸都不一样，后期没法锁 |
| **身材 / 比例** | 身高、胖瘦、头身比倾向、肩颈特征 | 不写→忽高忽矮忽胖忽瘦 |
| **气质 / 性格** | 温柔/冷峻/阳光/忧郁——这是模板明确要求提炼的关键词 | 不写→表情木讷、没有"人味" |
| **身份 / 年龄** | 学生/上班族/医生/古代将军 + 大致年龄 | 不写→着装乱来 |
| **基础着装** | 该身份最自然的"日常默认态"服装 | 不写→AI 随机给衣服，衍生换装时基线乱 |

> 注意一个画风强约束：`art_character.md` 里写明"**禁止内衣打底**""基础服装是日常默认态""焦点在面容与体态"。所以**基础角色不要写华丽礼服/战斗装**，那些是衍生层的事。基础层只写"这个人素颜、穿日常衣服时长什么样"。

#### Before / After 范例

**范例 1 · 女主（差）**

```
describe: 漂亮的女生，长头发，很温柔
```

problem：没有五官锚点、没有身高体型、没有身份、没有具体着装。"漂亮""长头发"信息量约等于零，AI 每次随机发挥。

**范例 1 · 女主（好）**

```
describe: 林晚秋，18岁高三女生。鹅蛋脸，丹凤眼，眼神温柔略带倦意，
鼻梁清秀，薄唇，左眼下方有一颗小泪痣。身高162cm，体型纤细偏瘦，
肩线柔和。黑色直发及腰，发质柔顺。气质安静、内敛、有书卷气。
日常着装为深蓝色90年代款式高中校服（白衬衫+深蓝背心裙），干净整洁。
```

为什么好：泪痣是**强识别特征**（模型最容易复现的锚点之一）；身高+体型给了比例约束；"安静、内敛、书卷气"被模板提炼成气质关键词写进 prompt；校服明确了基础着装。这一张底模出来，后面换装锁脸就稳了。

**范例 2 · 男主（差）**

```
describe: 帅气的霸道总裁，西装
```

problem：现代都市画风还好，但放到日系/古风画风里"霸道总裁"会让 AI 困惑；西装是着装但没说脸。

**范例 2 · 男主（好）**

```
describe: 沈彻，28岁，集团少东家。剑眉星目，眼神锐利沉稳，
高鼻梁，唇形利落，下颌线分明。身高184cm，宽肩，体型挺拔健硕。
黑色短发利落，鬓角干净。气质冷峻、强势、带距离感。
日常着装为合身的深灰色商务西装，无花纹，质感利落。
```

为什么好：五官给了"剑眉星目/锐利"这种气质同步的描述；身高 184 + 宽肩约束男性头身比（模板默认男性 6.5–7.5 头身）；"冷峻、强势"是清晰的气质词。

**范例 3 · 配角老人（差 vs 好）**

```
差: 一个老爷爷
好: 周明远，65岁退休中学教师。方脸，浓白眉，眼神慈和，法令纹明显，
   花白短发，戴一副金丝圆框眼镜。身高170cm，微胖，背略驼。
   气质温厚、儒雅。日常着装为深棕色针织开衫+衬衫，朴素。
```

老人尤其依赖"年龄特征词"（法令纹、花白、略驼），否则 AI 会画成"中年人 + 白发"的违和感。

### 2.2 衍生角色：**只写变化量**

这是最多人踩坑的地方。衍生角色（`assetsId` 指向基础角色）的 describe，**绝对不要重复写长相**，只写"相对底模变了什么"。

原因看代码就懂——衍生出图时（`batchGenerateAssetsImage.ts`），系统做了三件事：

1. 取父级的 `describe` 作为"父级资产描述"；
2. 取本资产的 `describe` 作为"当前资产描述"；
3. **把父级那张图的 base64 当参考图传给图像模型做 img2img**（`referenceList: [{type:"image", base64: 父级图}]`）。

也就是说，脸是靠**父级图 img2img 锁定**的，不是靠你在衍生 describe 里再描述一遍脸。而衍生用的模板 `art_character_derivative.md` 第一条铁律就是"**面容不变**——叠加后五官必须与底模完全一致，禁止面容偏移"，并把衍生范畴严格限定在 L1–L6（妆容/发型/服饰/鞋履/配饰），**禁止改姿态、禁止加场景、禁止加道具**。

所以你在衍生 describe 里重复写脸，等于给模型两个相互打架的信号：参考图说"用这张脸"，文字又说"丹凤眼薄唇小泪痣……"——文字描述会把模型从参考图上拽走，**这正是"漂脸"的头号成因**。

#### Before / After 范例

**范例 4 · 女主换"宴会盛装"（差）**

```
describe: 林晚秋，鹅蛋脸丹凤眼，身高162cm纤细，黑色长发，
穿一身红色晚礼服参加宴会，化了精致的妆
```

problem：又把脸、身高、发色全写了一遍。和父级图 img2img 信号冲突，脸会飘。

**范例 4 · 女主换"宴会盛装"（好）**

```
describe: 正式宴会造型。酒红色及地丝绒晚礼服，露肩款，质感垂坠。
发型改为低盘发，配珍珠发饰。精致正式妆，眼妆明显，唇色饱满。
银色细高跟鞋，珍珠耳环+项链。
```

为什么好：通篇没提一个字的五官——把"锁脸"完全交给 img2img；只描述了 L1 妆容、L2 发型、L4 主服、L5 鞋、L6 配饰。模板会照着这些层叠加，脸纹丝不动。

**范例 5 · 男主换"居家休闲"（好）**

```
describe: 居家休闲造型。浅灰色针织开衫内搭白色棉T，深色家居长裤，
发型放松微乱，伪素颜（看着没化妆但气色好）。室内棉拖鞋。无配饰。
```

**范例 6 · 女主"受伤虚弱"状态（好）**

```
describe: 病弱状态。面色苍白，唇色极淡，眼妆极轻（病弱妆）。
发型为松散的低马尾，几缕碎发垂落。穿浅米色宽松居家针织衫。
```

> 这里对应模板的"线索到妆容映射"：写"面色苍白、唇色极淡"会被识别为**病弱妆**线索。你不需要懂妆造术语，用**剧情态的自然语言**描述即可，模板的决策层会翻译。

### 2.3 describe 写法速记

| 资产层级 | 该写 | 不该写 |
|---|---|---|
| 基础角色 | 五官+身材+气质+身份+**日常**着装 | 礼服/战斗装/特定场景妆容 |
| 衍生角色 | **只写变化量**：妆/发/衣/鞋/配饰 | 任何五官/身高/发色等长相描述、场景、道具、姿态动作 |

## 三、资产类型的 describe 写法差异

`type` 决定系统用哪份模板（见 `batchGenerateAssetsImage.ts` 第 40–53 行：role 用 `art_character_derivative`，tool 用 `art_prop_derivative`，scene 用 `art_scene_derivative`）。不同类型的描述重点完全不同：

| 类型 | 模板 | describe 写作重点 | 关键约束（来自模板） |
|---|---|---|---|
| `role` 角色 | `art_character*` | 五官/身材/气质/身份/着装（基础）；妆发服饰（衍生） | 基础禁内衣打底；衍生面容不变、禁场景禁道具禁改姿态 |
| `scene` 场景 | `art_scene*` | 室内/室外、场景类型、季节+时间、前/中/后景元素、氛围 | **严禁出现任何人物、人影、人体轮廓**；必须有纵深层次；室外必须有空气透视；单画面非拼图 |
| `tool` 道具 | `art_prop*` | 道具名称、材质、形态、年代感、与角色/剧情的关系 | 单体展示、风格统一、与世界观时代感一致 |
| `clip` | —（无对应衍生模板，会回退） | 特殊用途，一般非手填 | — |

**场景 describe 范例（好）：**

```
describe: 室内·女主的卧室。90年代日式风格小房间，秋日午后。
前景：靠窗的木质书桌+台灯+摊开的课本；中景：单人床铺浅蓝格子床单、
墙上贴着海报；后景：半开的窗、窗外暖橙色夕阳与远处屋顶。
暖色调，柔和电影光从窗口斜射进来。家具有自然使用痕迹。
```

为什么好：明确了室内、季节时间、三层纵深、光线方向。模板会强制"无人物"，所以**不要在场景里写任何人**。

**道具 describe 范例（好）：**

```
describe: 一枚旧怀表。黄铜外壳已氧化发暗，表盘罗马数字，
表链略有磨损，背面刻有花纹。是男主父亲的遗物，承载回忆。
```

> 坑：很多人把场景 describe 写成"女主站在卧室里看书"。场景模板会**强行去掉人物**，但你这句话会污染前/中/后景的提示词，导致构图错乱。**场景只写空间，人交给分镜。**

## 四、角色一致性原理：脸到底是怎么锁住的

把链路串起来看一遍（`batchGenerateAssetsImage.ts`）：

```
基础角色出图：
  describe(全套长相)
    → AI(system = art_character 模板, user = describe)
    → 生成 prompt → 图像模型出图（无参考图或弱参考）
    → 落库为底模图 filePath

衍生角色出图：
  系统取 父级.describe + 本级.describe
    → AI(system = art_character_derivative 模板,
          user = "父级资产描述:{父级describe}\n当前资产描述:{本级describe}")
    → 生成 prompt
    → 图像模型 run({ prompt, referenceList:[{image: 父级图base64}], ... })  ← img2img 锁脸
    → 落库
```

**两道保险共同锁脸：**

1. **文字保险**：父级 describe 进了 system 的 user 输入，让生成的 prompt"知道这个人原本长什么样"；衍生模板反复强调"面容与底模一致"。
2. **图像保险（更强）**：父级那张底模图以 base64 形式作为 `referenceList` 传给图像模型做 img2img。这是像素级的脸部参考，比任何文字都准。

### 为什么"在衍生里重复写长相"会漂脸

img2img 的工作方式是：以参考图为起点，按 prompt 做修改。如果你的 prompt 里又出现大段五官描述，模型会把它当成"需要重新生成的脸"的指令，于是它会**偏离参考图去重画脸**——结果就是越描越不像。正确做法是让 prompt 只谈"要改的层"（妆发衣鞋饰），脸的部分**留白**，模型自然会忠实于参考图。

### 不一致问题排查顺序

1. **先看底模图本身好不好。** 底模脸如果就模糊/有瑕疵，后面全废。底模不行先重出底模。
2. **看衍生 describe 有没有混入长相描述。** 有就删干净，只留变化量。
3. **看父级图是否成功传入。** 看 `o_image` 里父级资产的 `state` 是否"已完成"、`filePath` 是否存在——代码里 `imageBase64` 来自 `imageUrlRecord[item.assetsId]`，**父级没出图或出图失败，参考图就是空数组**，img2img 退化成纯文生图，必飘。
4. **看 `assetsId` 指对了没。** 父子关系挂错，参考图就是别人的脸。
5. **看图像模型是否支持 img2img / 多图参考。** 换个 `imageModel` 试。

## 五、"Prompt 重新生成"依赖链：想让画面更合理，先动哪里

完整依赖链（自上而下，前者影响后者）：

```
画风选择(项目.artStyle)
  └─ prefix.md（全局美学：色盘/色温/必守/严禁，所有技能继承）
       └─ art_character[_derivative].md（角色专用约束模板）
            └─ 父级.describe + 本级.describe（你填的）
                 └─ AI 生成 prompt（写回 o_assets.prompt）
                      └─ 图像模型(imageModel) + 质量(imageQuality) + 比例
                           └─ 最终图
```

加载逻辑见 `getArtPrompt.ts`：它把 `prefix.md` 的内容拼在指定模板（如 `art_character_derivative.md`）前面，组成完整 system 提示词。所以**每张图的 system 永远 = prefix + 对应模板**。

**调优收益排序（先调收益高、成本低的）：**

| 优先级 | 调什么 | 成本 | 收益 | 适用场景 |
|---|---|---|---|---|
| 1️⃣ **describe** | 改资产描述，重新生成 | 极低 | 极高 | 绝大多数问题先动这里。脸不对、衣服不对、缺细节 |
| 2️⃣ **art_character\*.md** | 改该画风的角色模板 | 中 | 高（影响**该画风所有角色**） | 系统性问题：所有角色都裁切头顶、都缺背景色、都不写气质 |
| 3️⃣ **prefix.md** | 改全局美学前缀 | 中 | 很高但**牵一发动全身** | 整体色调/色温/风格基线要改。影响该画风**所有类型** |
| 4️⃣ **模型 / 质量** | 换 `imageModel` / 调 `imageQuality` | 低（配置项） | 视模型而定 | 前三步都对了但模型本身能力不行 |

> 改 skill 文件（2/3）**即时生效、无需重新构建**（见 `CLAUDE.md`："编辑 skill 文件立即生效"）。但请记住：改模板/prefix 是"批量手术"，先在一两个角色上验证再推广，别一上来就改 prefix 把全剧画风带偏。

> 工程师提醒：模板里那条 `R8 仅输出提示词——禁止输出表格/方案/解释` 很重要。如果你改模板时不小心让 AI 输出了解释性文字，这些垃圾会混进 `prompt` 字段直接喂给图像模型，导致画面里出现莫名其妙的元素或文字。改完模板一定回头看一眼 `o_assets.prompt` 生成得干不干净。

## 六、imageQuality 与 aspectRatio 的取舍

两者都来自 `o_project` 设置（`imageQuality` / `videoRatio` / `artStyle`），出图时透传给图像模型。

### imageQuality：1K / 2K / 4K

| 档位 | 适用 | 代价 |
|---|---|---|
| **1K** | 草稿期、批量试角色/试画风、快速迭代 describe | 速度快、便宜；细节（发丝、五官、布料纹理）会糊，**不要用 1K 出底模再去锁脸** |
| **2K** | **推荐默认**。角色底模、衍生资产、大多数分镜 | 速度/质量/成本平衡点 |
| **4K** | 关键海报、需要大特写的镜头、终稿 | 慢、贵；并发高时更容易超时失败 |

**实战建议：** 试错阶段用 1K 快速跑通 describe，定稿后**底模和衍生统一上 2K**（底模清晰，img2img 锁脸才稳），个别关键镜头 4K。

### aspectRatio：比例

代码里有两个不同来源，注意区别：

- **资产图**（`batchGenerateAssetsImage.ts`）：硬编码 `aspectRatio: "16:9"`。因为角色是**四视图设定图**，模板建议 4:1 / 3:1，但实际出图走 16:9 横幅来容纳并排四视图。这个你改不了（除非改代码）。
- **分镜图**（`batchGenerateImage.ts`）：用项目的 `videoRatio`。**这才是你要关心的成片比例。**

| 比例 | 适用 |
|---|---|
| **16:9** | 横屏（B站/YouTube/横版漫剧） |
| **9:16** | 竖屏短视频（抖音/快手/Reels）——短剧主流 |
| **1:1 / 4:5** | 部分社交平台 |

> 坑：`videoRatio` 决定分镜构图，**一旦定了就别中途改**。改了之后已生成的镜头比例不一致，混剪会很难看。开项目前先想清楚是横屏还是竖屏。

## 七、分镜出图：让镜头里的人和景一致

分镜出图入口 `src/routes/production/storyboard/batchGenerateImage.ts`。和资产出图最大的不同：**分镜的 prompt 不在这里生成**（它来自分镜面板已有的 `o_storyboard.prompt`），这里只做"带参考图出图"。

### 关联资产如何作为参考图

链路（代码第 51–78、142–169 行）：

1. 从 `o_assets2Storyboard` 按 `rowid` 顺序查出该镜头关联的所有 `assetId`（角色、场景、道具）——**顺序有意义，按入表顺序排**；
2. 用 `assetId` 查到各自的 `imageId`，再查 `o_image.filePath`；
3. 把这些图转成 base64，组成 `referenceList` 传给图像模型。

也就是说：**这个镜头要保持一致的角色和场景，必须先在分镜里"关联"上对应资产**，而且那些资产**必须已经成功出过图**（`filePath` 存在）。代码里 `getAssetsImageBase64` 会静默跳过取不到图的资产（`return null` 然后 filter 掉）——所以**关联了但没出图的资产，等于没参考图**，那个角色在镜头里就会"陌生脸"。

### 怎么保证镜头里角色/场景一致

| 做法 | 说明 |
|---|---|
| **先把资产全部出好图，再批量出分镜** | 顺序不能反。资产没图，分镜就没参考 |
| **每个有人物的镜头都关联对应角色资产** | 漏关联 = 该角色在这镜里随机长相 |
| **关联场景资产** | 保证同一地点跨镜头一致 |
| **关联顺序保持稳定** | 多角色镜头里，参考图顺序按 `rowid`，影响模型对"谁是谁"的理解 |
| **优先关联衍生资产而非基础底模** | 如果这镜里女主穿的是"宴会装"，就关联那个衍生资产的图，而不是穿校服的底模 |

### 出图失败如何重试

`batchGenerateImage.ts` 用了一个 `compulsory`（强制）参数控制重试逻辑：

- **`compulsory = false`（默认）**：只出 `shouldGenerateImage = 1` 的镜头；`shouldGenerateImage = 0` 的标记为"未生成"跳过。用于常规批量出图。
- **`compulsory = true`**：把指定镜头全部强制标记为"生成中"+`shouldGenerateImage=1` 并重出，**无视 shouldGenerateImage 开关**。这是你"重试失败镜头/强制重画"的开关。

失败的镜头在 `o_storyboard` 里 `state="生成失败"`，失败原因写在 `reason` 字段。重试步骤：

1. 找出 `state="生成失败"` 的 `storyboardIds`；
2. 看 `reason`（超时？模型报错？参考图缺失？）；
3. 若是偶发超时——直接带 `compulsory:true` 重试；
4. 若是参考图缺失——先去把关联资产出好图，再重试；
5. 高并发批量出图时超时率高，可**调小 `concurrentCount`**（默认 5）。

> 并发坑：`concurrentCount` 默认 5（资产和分镜都是）。4K + 慢模型 + 并发 5，很容易整批超时失败。批量 4K 时建议把并发降到 2–3。

## 八、排障表：症状 → 病因 → 改哪里

| 症状 | 最可能的病因 | 改哪里（按先后） |
|---|---|---|
| **漂脸**：衍生/分镜里角色不像底模 | ① 衍生 describe 重复写了长相，和 img2img 打架 ② 父级图没出好/没传进去 ③ `assetsId` 挂错 ④ 模型不支持 img2img | ① 删掉衍生 describe 里所有五官/身高/发色描述，只留变化量 ② 查 `o_image` 父级 `state`/`filePath` ③ 核对父子关系 ④ 换 `imageModel` |
| **画风跑偏**：出来不像选定画风 | ① describe 里写了与画风冲突的词（如日系画风写"霸道总裁/赛博朋克"）② prefix.md 被改坏 ③ 模型本身画风倾向强 | ① 清理 describe 里的冲突词，用画风自带语汇 ② 检查 `prefix.md`（注意它的"严禁项"）③ 换模型 |
| **出现不该有的人物**（场景图里有人） | 场景 describe 里写了人物/动作 | 场景 describe 只写空间，删掉所有人物相关描述（场景模板本应去人，但描述会污染 prompt） |
| **构图错乱 / 拼图 / 多视图叠在一起** | ① 场景 describe 暗示了多视图 ② 分镜 prompt 里残留模板解释性文字 ③ 比例与内容不匹配 | ① 场景写"单画面" ② 检查 `o_storyboard.prompt` 是否干净，必要时改模板的 R8 约束 ③ 核对 `videoRatio` |
| **角色在镜头里成了陌生脸** | 该镜头没关联对应角色资产，或关联的资产没出图 | 在分镜里关联角色资产，并确认该资产 `o_image.state="已完成"` |
| **同一地点跨镜不一致** | 镜头没关联场景资产 | 给同地点的镜头都关联同一个场景资产 |
| **细节糊（发丝/五官/布料）** | `imageQuality` 太低 | 底模/衍生升到 2K，关键镜头 4K |
| **批量整片失败（多为超时）** | 并发太高 + 高分辨率 + 慢模型 | 调小 `concurrentCount`，降到 2–3；`reason` 字段确认是否超时 |
| **prompt 里混进解释文字/表格** | 模板被改后 AI 输出了非提示词内容 | 检查对应 `art_*.md` 的"仅输出提示词"约束，重生 prompt |
| **所有角色都同一个问题**（都裁头顶/都缺气质/都同款衣服） | 这是模板级问题，不是单个 describe | 改 `art_character*.md`（影响该画风全体），先验证再推广 |
| **衍生角色姿态/场景乱跑** | 衍生 describe 里写了动作/场景/道具 | 删掉，衍生只允许 L1–L6（妆发衣鞋饰） |

## 本章调优速查表

**写 describe（90% 的问题在这）**

- 基础角色：五官 + 身材 + 气质 + 身份 + **日常**着装，写全；加一个强识别特征（痣/疤/眼镜）。
- 衍生角色：**只写变化量**（妆/发/衣/鞋/饰），**一个字都别提脸**。
- 场景：只写空间（室内外/季节时间/前中后景/光线），**绝不写人**。
- 道具：材质 + 形态 + 年代感 + 与剧情的关系。

**一致性**

- 脸靠"父级图 img2img"锁，不靠文字 → 衍生里别复述长相。
- 漂脸先查：底模质量 → 衍生 describe 是否混入长相 → 父级图是否出成功 → `assetsId` 是否挂对。

**调优顺序（收益从高到低）**

```
describe  >  art_character*.md  >  prefix.md  >  模型/质量
（即时生效，但模板/prefix 是批量手术，先验证再推广）
```

**质量与比例**

- 草稿 1K，定稿底模+衍生统一 2K，关键镜头 4K。
- `videoRatio` 开项目前定死（竖屏短剧 9:16 / 横版 16:9），中途别改。
- 资产图比例固定 16:9（四视图），改不了；分镜走 `videoRatio`。

**分镜**

- 先出全部资产图，再出分镜；每个有人物的镜头都关联对应角色资产，同地点关联同一场景资产。
- 镜头里换了装就关联**那个衍生资产**，不是底模。
- 重试用 `compulsory:true`；批量超时就把 `concurrentCount`（默认 5）降到 2–3。
