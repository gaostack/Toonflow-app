> 这是一套写给**创作者**(而不是程序员)的体系化使用手册——目标是让你像一位带着制片人、编剧、导演、美术、后期经验的"剧组总指挥"那样,把 Toonflow 这台"小说→漫剧"的自动化机器**调到你想要的样子**。

## 一、先建立一个核心心智模型:你只有两条"调优通道"

无论你想让作品在哪个环节变得更好,落到操作上永远只有两个着力点。**分清它们,是用好这个工具的全部前提。**

```
        ┌──────────────────────────────────────────────┐
        │  你想要的最终效果(更爽的剧情 / 更对的画风 /     │
        │   更有张力的镜头 / 更稳的角色一致性 / …)         │
        └───────────────┬──────────────────────────────┘
                        │ 落地为两件事
        ┌───────────────┴───────────────┐
        ▼                               ▼
┌─────────────────────┐       ┌──────────────────────────────┐
│ ① 项目字段(选择题)   │       │ ② 技能文件 .md (标准答案)       │
│ 创建项目时填的信息    │       │ data/skills/ 下的提示词         │
│ 决定"用哪套手册/模型" │       │ 决定"AI 怎么想/怎么写/怎么画"   │
│ 改了即时生效          │       │ 改文件即时生效,无需重启/编译    │
└─────────────────────┘       └──────────────────────────────┘
```

> 🔑 **黄金法则:大多数"质量问题"应该改 ② 技能文件,而不是反复点"重新生成"。** 输入不变,AI 的上限就不变。重试只在"已经调好规则、只是想换一版发挥"时才有意义。

并且记住三条贯穿全书的事实:

1. **技能文件即时生效**——`data/skills/` 下任何 `.md` 改完保存,下一次生成就用新内容,不用重启服务、不用重新打包。(Docker 部署时注意:`data/` 是持久卷,改动要落在挂载目录里。)
2. **记忆 ≠ 产物**——清空对话记忆只会让 AI"忘记你们聊过什么",**不会删除已生成的骨架/剧本/角色/分镜**。
3. **画面问题不要去清记忆**——画面由"视觉手册 + 角色 describe + 模型"决定,和对话记忆无关。

## 二、一张图看懂整条流水线 & 各环节归谁管

```
小说原文
  │  ① 项目字段:type / intro / artStyle / directorManual / 模型 / mode …
  ▼
【导入&事件抽取】 o_novel(整章原文 + 自动抽取的"事件表")     → 第02章
  ▼
══════════ 剧本 Agent(吃:小说名/类型/简介/视觉手册/画幅/章节数)══════════
  │  故事骨架 → 改编策略 → 逐集剧本                              → 第03章
  ▼
══════════ 制作 Agent(吃:视觉手册+导演手册技能 / 模型信息)══════════
  │  导演规划 → 衍生资产分析 → 资产出图 → 分镜表 → 分镜面板 → 分镜图
  │     ↑ 系统提示词骨架(production_*.md)                      → 第06章
  │     ↑ 视觉手册肌肉(art_skills)                             → 第04章
  │     ↑ 导演手册肌肉(story_skills)                           → 第05章
  │     ↑ 角色/资产出图 prompt 工程(describe + 模板 + img2img)  → 第07章
  ▼
【视频提示词 → 视频生成(mode!) → 配音 → 选片 → 成片】           → 第08章
  ▼
最终漫剧短视频
```

> 注意 **`directorManual`(导演手册)不进剧本 Agent,只进制作阶段**;而 `type/intro`(类型/简介)只进剧本 Agent。所以:剧情不对味 → 调类型/简介 + 剧本类技能;镜头/画面叙事不对味 → 调导演手册。

## 四、怎么用这套手册(三类读者的路线)

- **第一次上手** → 按 00 → 01 → 02 → 10 顺序读,先用第10章的"配方"跑通一部,再回头精修。
- **某个环节不满意,想精修** → 直接跳对应章的"本章调优速查表"和"症状→病因→改哪里"排障表。
- **想做出独家风格/题材** → 精读第04章"新建画风"与第05章"新建题材",复制内置目录改造。

每一章都遵循统一结构,方便检索:
> **本章导览** → 逐项调优(控制什么/创作意义/怎么调/before-after/常见错误)→ 对照表 → **本章调优速查表**。

## 五、最重要的几条"老法师"经验(读之前先记住)

1. **简介(intro)是性价比最高的旋钮。** 别写营销语("超爽神作"),要写真实主线 + 核心爽点 + 人物钩子。它直接喂进剧本 Agent。(详见第01、03章)
2. **角色 describe 是出图性价比最高的旋钮,而且零成本。** 基础角色写全,衍生角色只写"变化量",否则会和 img2img 打架导致漂脸。(详见第07章)
3. **先把"基础角色"调满意,再批量生成衍生。** 衍生继承父级,父级不好,全盘皆输。
4. **视频模型(videoModel)和 mode 必须自洽。** 选错会导致提示词模板不匹配、参考图白送、算力空烧。(详见第08章)
5. **改"骨架级"系统提示词(`production_*.md`)要谨慎**——容易破坏流水线串行与 XML 输出格式;调"肌肉"(视觉/导演手册)更安全。(详见第06章)
6. **量化阈值改了要"成对同步"**——比如分镜表的时长阈值若与监督层维度不一致,会出现"生成了又被打回"的拉锯。(详见第03、06章)
7. **遇到卡顿先"重新连接",不要动不动"清空全部记忆"。** (详见第09章)
